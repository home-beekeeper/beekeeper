What it does

Stop supply chain attacks before they reach your agents.

Beekeeper sits between your coding agent and your machine. Every install, file read, and network call passes through a policy engine that knows what attackers do.

● Hook into every agent

First-class integration for Claude Code, Cursor, Codex CLI, OpenCode, Continue, and any MCP-speaking client. Tool calls cannot bypass the harness.

● Corroboration-based matching

Catalog source diversity is the 2FA principle for threat intel. One source warns. Two sources enforce. Three sources quarantine.

● Real-time process correlation

Sentry watches kernel events for the exfiltration signature: extension host reads credentials, opens a connection. Fires regardless of what catalogs know.

● Calm-mode dashboard

TUI that stays quiet by design. The base screen escalates on its own when something fires, with containment actions one keystroke away.

● Steer toward hardened tooling

When an agent runs npm install, Beekeeper nudges to pnpm or Bun if installed. These ship structural defenses npm doesn't have.

● Open from day one

Apache 2.0. Reproducible builds. Sigstore-signed releases. SLSA L3 provenance. No telemetry, no upsell, no SaaS lock-in.

How it works

Two layers, working together.

Reactive defense catches what's been catalogued. Proactive defense catches what hasn't been. Each layer compensates for the other's blind spots.

Layer 1 / Reactive

Match every install against intelligence

Catalogs sync hourly from Bumblebee, OSV, and Socket. When a new entry lands for a package on your machine, Beekeeper flags it within minutes.

The 2FA principle keeps false positives bounded: a single source can only warn, never enforce. Real blocks require two independent catalogs to agree.

Best at catching: known campaigns, named packages, post-disclosure threats.
Layer 2 / Proactive

Watch behavior, not just identity

Sentry runs as a privileged daemon. It correlates process events, file access on sensitive paths, and outbound network connections into the exfiltration signature attackers can't avoid expressing.

Fires when an extension host descendant reads credentials and opens a connection. The Nx Console pattern, expressed in behavior.

Best at catching: zero-day campaigns, novel malware, anything pre-disclosure.

FAQ

Common questions.

Is Beekeeper a replacement for Snyk, Socket, or Dependabot? +

No. Those tools answer "is this package vulnerable according to its CVE database?" That question is reactive by design and tuned for human-driven workflows. Beekeeper answers "did an autonomous agent on my machine just attempt something an attacker would do?" That question needs real-time interception at the tool-call layer. They complement each other. Many teams will want both.

Does Beekeeper send my code or data anywhere? +

No. The audit log writes to disk locally by default. Optional sinks (syslog, OpenTelemetry, HTTPS) exist for teams that want centralized logging, but they require explicit configuration. Beekeeper has no built-in telemetry, no phone-home, no analytics.

What's the performance cost? +

Hook handler latency targets sub-100ms p99 per tool call. Sentry daemon (optional, opt-in via beekeeper protect install) averages 0.5-3% of one CPU core in steady state, with 200MB resident memory. Lighter than Slack desktop. Detailed envelopes in the runtime model docs.

Does it work on Windows? +

Yes, fully. Cross-platform parity (macOS, Linux, Windows) was a v1.0 ship requirement. Beekeeper uses Pollen (our Windows-compatible inventory layer derived from Bumblebee) as the on-disk scanner.

What if my coding agent isn't supported? +

Any agent that speaks the MCP protocol can route through Beekeeper's gateway daemon without first-class integration. For agents that don't, a shim layer wraps npm, pnpm, pip, cargo, gem, and composer to catch installs at the shell layer.

Can I trust the binary I just downloaded? +

Verify it yourself. Every release ships with Sigstore signatures, SLSA L3 provenance attestation, and reproducible build instructions. The make verify-release target reproduces the binary from source and compares hashes against the published artifact. The project's stance is that distrust is the appropriate posture. Don't trust us; verify us.

What's the license? +

Apache 2.0. Use it, fork it, ship it inside your product, do whatever the license permits. No CLA required to contribute.
