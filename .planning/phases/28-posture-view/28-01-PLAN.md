# Plan 28-01 — `beekeeper posture` read-only view

## Goal (PRD Layer 2)

A read-only `beekeeper posture` view (CLI + TUI) that reads each detected package
manager's actual config and shows it side by side with Beekeeper's enforced
posture, naming the gaps Beekeeper covers. Machine-wide. It NEVER writes any
package-manager config file. The enforcement-boundary statement appears in the
output.

## Context (reuse, do not rebuild)

- `internal/posture` already has the read-only detectors (relocated from the
  nudge in Phase 26): `DetectState(ctx, Config) PMState` (pm versions + hardening),
  `DetectPnpmHardening(path) HardeningResult` (reads pnpm-workspace.yaml
  minimumReleaseAge), `DetectBunScanner(paths) bool` (reads bunfig.toml), and the
  `PMState` type. These ONLY read; they never write. Read `state.go` for the exact
  PMState fields.
- Beekeeper's enforced default posture: release-age 1440m/24h warn
  (`policy.DefaultReleaseAgeConfig`), lifecycle-script warn, git/remote-URL warn.
  Per-rule action defaults to warn (the opt-up-to-block config is Phase 29; this
  view shows the default actions and can note they are tunable in Phase 29).
- The canonical boundary statement: `posture.BoundaryStatement` / `BoundaryShort`.

## Tasks

### Task 1 — the posture model (internal/posture)
Add a pure function that, given a detected `PMState` (+ the read config hardening
results), produces a structured comparison: for each detected ecosystem/manager,
what the manager's own config does by default vs what Beekeeper enforces, and the
GAPS Beekeeper covers. Example rows the model must make legible (PRD):
- "npm v11 detected. Scripts run by default. Git deps allowed. Beekeeper enforces:
  scripts warned, release-age 24h, git deps flagged. Covering 3 gaps your npm
  version does not."
- "pnpm v11 detected. minimumReleaseAge honored. Beekeeper enforces: aligned, no gap."
Keep the comparison logic pure/testable (no I/O); the I/O (DetectState + scanners)
is done by the caller and passed in.

### Task 2 — the CLI command (cmd/beekeeper)
`beekeeper posture` (new subcommand, registered in newRootCmd):
- Resolve config (resolveConfig) + run posture.DetectState + the scanners
  (read-only) to gather each pm's state.
- Render the comparison table (the Task 1 model) to stdout: detected managers,
  their config, Beekeeper's enforced posture, and the named gaps.
- Print `posture.BoundaryShort` (or BoundaryStatement) so the honesty boundary is
  visible: this is what Beekeeper ENFORCES at the hook; it is not a claim about
  what your package managers do at the machine level.
- READ-ONLY: this command must never write a pm config file. Add a short Long
  description stating it reads config and never modifies it.

### Task 3 — the TUI panel (internal/tui)
Add a read-only posture panel to the dashboard (mirror an existing read-only panel
like the catalogs panel): shows the same comparison, no admin actions, no writes.
Reuse the Task 1 model so CLI and TUI share one source of truth. Match the
existing Bubble Tea v2 panel patterns (charm.land/bubbletea/v2).

### Task 4 — tests
- Pure comparison-model tests: npm-with-scripts-on → 3 gaps; pnpm-with-minimum-
  release-age → aligned/no gap; an undetected/absent manager is omitted. Assert the
  named gaps and the "covering N gaps" count.
- A read-only guarantee test: drive the CLI/scan path against a temp dir holding
  fixture pm config files (.npmrc / pnpm-workspace.yaml / bunfig.toml) and assert
  the files are byte-for-byte UNCHANGED after `beekeeper posture` runs (IPVIEW-02 —
  never writes config). This is the load-bearing self-defense test for Layer 2.
- A CLI-output test: covers npm AND one other ecosystem (pnpm), asserts both appear
  with their gaps and that BoundaryShort/Statement text is present (IPBND-01).
