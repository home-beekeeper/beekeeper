<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| build-time source → public sitemap | sitemap.ts enumerates Fumadocs sources at build time; the resulting out/sitemap.xml is world-readable once deployed. |
| BASE_URL constant → every <loc> | All sitemap/robots URLs derive from the locked BASE_URL (17-01). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-02 | Information Disclosure | `out/sitemap.xml` + `out/robots.txt` | mitigate | sitemap enumerates ONLY public content routes via `source.generateParams()` (no 404/_not-found, no `~/.beekeeper`, no build-internal or private paths). robots.txt exposes only an allow-all rule + the public sitemap URL. The 17-01 seo_spec.py harness asserts EVERY `<loc>` starts with BASE_URL — a leaked private/relative path fails the build gate. |
| T-17-03 | Spoofing | sitemap/robots host (BASE_URL) | accept | URLs derive from the hardcoded BASE_URL constant (17-01), not user input — no host-injection / open-redirect surface. |
| T-17-05 | Denial of Service | `force-static` export under output:'export' | mitigate | Both route handlers set `export const dynamic = "force-static"` so they prerender at build time (no runtime route execution) — consistent with fail-static; a missing force-static fails the build loudly (caught by the build-gate), it does not silently ship a broken/serverful route. |
| T-17-SC | Tampering | npm/pip/cargo installs | accept | NO new packages installed (uses already-installed next@16.2.7 MetadataRoute API). No package-legitimacy checkpoint needed — see 17-RESEARCH Package Legitimacy Audit. |
</threat_model>
