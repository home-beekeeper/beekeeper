<objective>
Fix the confirmed WPATH-02 defect in the **Pollen** emitter (repo `github.com/home-beekeeper/pollen`,
working tree `../pollen`, HEAD c94b271): `endpoint.Current()` assigns `ep.UID = u.Uid`
unconditionally (line 29), and the `user.Current()` error fallback assigns
`ep.UID = strconv.Itoa(os.Getuid())` (line 31). On Windows `user.Current().Uid` is a SID string
(`S-1-5-21-...`) and `os.Getuid()` returns `-1` — neither is empty. WPATH-02 requires `endpoint.uid`
to be the empty string on Windows while remaining a populated numeric UID on Linux/macOS.

Per RESEARCH.md Open Question 2 + Pitfall 5: use the inline `runtime.GOOS != "windows"` guard on
BOTH assignments (fewer files than build-tagged stubs, and a single-field override does not warrant a
file pair). All other endpoint fields (os, arch, username, hostname, device_id) are already correct
on Windows and must stay untouched.

Purpose: Satisfy WPATH-02 endpoint-record half (empty Windows uid, no SID leakage) without changing
the NDJSON schema (uid field stays present, just empty) and without altering Unix behavior (D-02, D-04).
Output: Two guarded assignments in endpoint.go and an extended endpoint_test.go asserting the
empty-on-Windows / non-empty-on-Unix contract plus the unchanged fields.
</objective>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Information Disclosure | endpoint.uid SID leakage on Windows | mitigate | Pre-fix, Windows emitted the user's SID (S-1-5-21-...) as `uid`, a host-identifying value with no inventory purpose. The fix sets uid to "" on Windows, eliminating SID presence in emitted records (RESEARCH Security Domain: SID-leakage / Information Disclosure). This is the security-positive intent of WPATH-02. |
| T-03-05 | Tampering | UID field on non-Windows (regression) | accept | The non-Windows code path is unchanged (numeric uid still emitted). The guard is a pure `runtime.GOOS` compile-time-constant comparison; no input is parsed. Regression is covered by TestCurrentWindowsUID's Unix branch. |
| T-03-06 | Information Disclosure | endpoint.username still emitted on Windows | accept | WPATH-02 explicitly requires a non-empty username on Windows; this is in-scope expected behavior of host-identity inventory, unchanged from current behavior on all platforms. No action. |

No HIGH-severity items. The single security-relevant change (T-03-04) is a disclosure REDUCTION (removing SID), not an addition of attack surface.
