## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Pollen NDJSON (subprocess stdout) → beekeeper internal/scan consumer | Untrusted-shaped NDJSON lines cross into beekeeper's Scan; malformed input must fail closed (scan_error) and valid input passes through |
| test fixture → parity/round-trip assertions | Hand-crafted Windows-shaped records exercise the consumer and parity paths (test-only, not production input) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-07 | Tampering | NDJSON injection via crafted backslash path in a scanned record | accept | beekeeper's Scan validates each line with json.Unmarshal(json.RawMessage) and emits scan_error (fail-closed) on malformed JSON before any passthrough. A backslash path is valid JSON content; it is written back verbatim with fmt.Fprintf, not interpreted as a filesystem path by the consumer. No new parsing or path resolution is added in this plan. The round-trip test confirms the fail-closed scan_error path is NOT triggered for a well-formed Windows record. |
| T-03-08 | Information Disclosure | endpoint.uid in the round-trip fixture | accept | The fixture intentionally carries uid="" (the WPATH-02-correct shape). No real SID is emitted; the test asserts the empty-uid shape survives. No disclosure. |
| T-03-09 | Tampering | release artifact integrity (VERSION/CHANGES) | mitigate (deferred) | Signed release (Sigstore/cosign + CycloneDX SBOM) is the integrity control for distributed artifacts; per D-06 it is batched to M2 close. This plan only prepares VERSION/CHANGES locally and explicitly forbids git-tag/sign, so no unsigned artifact is published. The integrity control is preserved, just deferred — matching the Phase-2 precedent. |

No HIGH-severity items. The consumer boundary (T-03-07) retains its existing fail-closed validity check; this plan adds only a test that exercises it, no new untrusted-input handling.
</threat_model>
