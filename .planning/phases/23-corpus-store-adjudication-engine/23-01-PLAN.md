<objective>
Stand up the append-only, owner-only, redaction-first corpus store (`StoreSink` as an `audit.Sink`) and the non-reversible HMAC-SHA256 fingerprint layer (`RepoFingerprint` / `FleetNodeID`) with a per-install salt persisted in `state.json`.

Purpose: This is the impure I/O foundation Phase 23 builds on. Plan 23-02 (emitter + envelope) and 23-03 (adjudicator + wiring) both import the real `StoreSink` and `RepoFingerprint`. Getting redaction-first, 0600 permissions, append-only, and HMAC non-reversibility correct here is what makes the corpus a safe place to persist incident data (PRD §3, F-1 security lesson).
Output: `internal/corpus/store.go`, `internal/corpus/fingerprint.go`, their Wave-0 test files, a new `DownstreamCleanDays` field on `config.CorpusConfig` (OQ-1), and a new `CorpusLocalSalt` field on `catalog.WatchState` (salt storage).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| AuditRecord → corpus NDJSON file | Untrusted tool-call-derived strings (Reason, target paths) cross into a persisted on-disk artifact |
| repo path / hostname → emitted fingerprint | Potentially identifying values are transformed before they can ever be transmitted (v1.1+) |
| agent process → corpus file on disk | A hijacked agent may try to read Beekeeper's detection knowledge or inject false records |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-23-01 | Information Disclosure | StoreSink.Write persisting credential-shaped strings | mitigate | `audit.RedactRecordWithDefaults` is the FIRST statement in `Write` (Task 2); `TestStoreRedactsSecretsBeforeWrite` proves AKIAIOSFODNN7EXAMPLE never lands on disk |
| T-23-02 | Information Disclosure | Reversible `repo_fingerprint` via dictionary attack on repo paths | mitigate | HMAC-SHA256 with a per-install random salt, never bare SHA-256 (Task 3); `TestFingerprintNonReversibility` proves salt-A ≠ salt-B |
| T-23-03 | Tampering | Agent writes/reads the corpus file to inject false adjudications or learn detection knowledge | mitigate | 0600 / Windows owner-DACL via `platform.SetOwnerOnly` re-enforced on every write (Task 2); O_APPEND only (no truncation); `TestStoreFilePermissions` |
| T-23-04 | Tampering | Corpus file path configured outside StateDir → self-protection guard (StateDir prefix only) does not cover it | mitigate | `ResolveCorpusPath` rejects any `cfg.Path` not under `stateDir` (Task 2, Pitfall 7) |
| T-23-SC | Tampering | npm/pip/cargo supply-chain | accept | No package installs in this plan — zero new go.mod entries (stdlib only: crypto/hmac, crypto/sha256, crypto/rand, encoding/hex, encoding/json, sync, os) |
</threat_model>
