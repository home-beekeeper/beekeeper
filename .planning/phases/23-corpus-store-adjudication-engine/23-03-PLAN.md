<objective>
Build the off-hot-path adjudication engine that assigns the outcome layer (pure `Adjudicate` + bounded `RunAdjudicationBatch`), wire the batch pass into `runCatalogsSync`, add the corpus `StoreSink` write to the `beekeeper check` chokepoint (fail-closed), and prove the hot path stays sub-100ms with corpus enabled.

Purpose: This completes the local loop. The store (23-01) captures every incident as `unresolved`; the adjudicator assigns ground truth (`malicious`/`benign`/`policy_correct`) via the six documented sources, gated by corroboration. Critically, all of this runs OFF the synchronous hook path: the corpus write is a fast synchronous NDJSON append inside the existing audit chokepoint, and adjudication only ever runs in `runCatalogsSync` (the OS-scheduler invocation path, OQ-3 locked — the `catalogs daemon` is a scheduler registrar, not a long-lived goroutine). A corpus write error must never change a block into an allow.
Output: `internal/corpus/adjudicator.go`, its tests, the `runCatalogsSync` batch-pass wiring, the check hot-path corpus write, the MultiSink corpus integration for daemon/gateway surfaces, and `BenchmarkRunCheck`.
</objective>
