<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| AdjudicationResult/runtime strings → PushEnvelope.action_hint | A push envelope is a future fleet-pushable artifact; its action_hint determines blast radius |
| catalog matches (some from a single source) → confidence_tier | A single compromised source must not drive enforce-weight |
| signing key on disk → agent | The Ed25519 private key must not be readable/forgeable by a hijacked agent |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-23-05 | Tampering (blast-radius) | `auto_purge` (or any non-allowlisted action) emitted in a push envelope | mitigate | `ActionHint` is a typed const (Phase-22 compile-time guard) + `BuildPushEnvelope` purge-class error return (Task 2) + `FuzzBuildPushEnvelope` ENV-03 gate (Task 3, release blocker). `auto_purge` is never a constructable value |
| T-23-06 | Tampering (bypasses 2FA) | Single-source critical escalation tagged `enforce` in the corpus | mitigate | `confidence_tier`/`source_count` derived ONLY from `policy.CorroborateOutcome` (count >= BlockAt), never from `level == "block"` (Task 2); `TestConfidenceTierTable` proves single-source-critical stays `watch` |
| T-23-07 | Spoofing | Forged signing block on an emitted envelope | mitigate | Ed25519 stdlib signer; private key generated once at 0600 / owner-DACL (Task 3); `Signing` nil in v1 until a key exists (no transport, so no live forgery surface yet) |
| T-23-08 | Information Disclosure | source_count re-derived downstream from event logs (double-count) | mitigate | `source_count` is FROZEN at emission in the envelope; consumers read it, never recount (ENV-02, Task 2) |
| T-23-SC | Tampering | npm/pip/cargo supply-chain | accept | Zero new go.mod entries — stdlib only (crypto/ed25519, crypto/rand, encoding/json, encoding/hex, fmt, time) |
</threat_model>
