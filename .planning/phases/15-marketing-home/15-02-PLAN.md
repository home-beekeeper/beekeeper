<feature_card_reconciliation>
D-03 + HOME-03 require the feature cards to cover EXACTLY these six SHIPPED capabilities, by
name (the executor maps the mockup's card styling onto this accurate content):

1. **Corroboration engine** — corroboration-based scoring: 1 source warns, 2 enforce/block,
   3 block + quarantine (README "Catalog-based threat intelligence"; CLAUDE.md locked decision).
2. **Fail-closed hooks** — PreToolUse hook runs `beekeeper check --hook <harness>` before each
   tool call; on block exits 2 + emits harness-specific deny JSON; fail-closed by default
   (README; CLAUDE.md "Fail closed by default").
3. **Editor-extension defense** — defends against the malicious VS Code / editor-extension class
   (the Nx Console attack vector); Phase-3 editor-extension defense is shipped.
4. **Sentry** — privileged runtime daemon (opt-in via `beekeeper protect install`) that correlates
   process + file + network events into the exfiltration signature; fires regardless of catalog
   knowledge.
5. **LlamaFirewall** — supervised Python sidecar (CodeShield + AlignmentCheck) for prompt-injection
   / unsafe-code detection; fail-closed on crash (CLAUDE.md locked decision).
6. **Policy-as-code** — declarative policy JSON files (`~/.beekeeper/policies/`) evaluated by the
   pure policy engine; layered config.

These six are the ONLY cards. Do NOT carry over the mockup's "Calm-mode dashboard",
"Steer toward hardened tooling" (npm→pnpm nudge), or "Open from day one" cards as the
canonical six — those are real features but they are not the six HOME-03 names. (You MAY mention
the nudge inside the policy-as-code or fail-closed-hooks card body as supporting prose, but the
six card TITLES are fixed to the list above.)
</feature_card_reconciliation>
