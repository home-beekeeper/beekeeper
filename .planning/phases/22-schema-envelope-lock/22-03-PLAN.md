## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| attacker behavior → behavior_signature_hash | The hash is a behavior fingerprint of the attacker; normalization must remove victim-specific context without losing attack specificity, and must not depend on machine state |
| re-scan → corpus incident identity | A re-scan of the same flagged version must map to the same cluster, not mint a duplicate incident (idempotency) |
| Nx Console trace → schema | The gate is the integrity boundary proving the schema can carry a real incident with no gaps before any record is written |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-22-10 | Tampering (data integrity) | BehaviorSigHash normalization | mitigate | Rules FROZEN + documented (lowercase/base-name action, home-prefix→~ target, port-stripped host); NUL separation prevents prefix collisions; no machine-dependent imports so the fingerprint is reproducible across victims (research Finding 4) |
