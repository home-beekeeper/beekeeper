<objective>
Tier 3 W1 — cheap Sentry rule wins on the EXISTING three event streams (no new
event source). Expand the credential watchlist (cloud harvesters: gcloud/azure/
kube/docker/.claude); add SENTRY-006 (agent-descendant credential cluster) by
refactoring the editor-only ancestry gate to a unified `isMonitoredDescendant`
(editor OR agent) — so agents in bare terminals/CI/SSH get coverage without VS
Code/Cursor integrated terminals double-firing; add SENTRY-007 (generalized exfil
fusion gated on an external destination via `isExternalDest`, dropping SENTRY-005's
fresh-extension precondition); and lock the rules layer's purity with a test.

Purpose: v1 Sentry is an editor-extension exfil detector blind to standalone-terminal
agents and cloud-cred harvesters. These four changes broaden coverage at near-zero
cost while keeping the engine pure (no I/O) per the CLAUDE.md constraint.

Output: expanded watchlist (+ daemon mirror); isAgentDescendant/isMonitoredDescendant/
isExternalDest; SENTRY-006/007; TestRulesImportsArePure.
</objective>
