<objective>
Tier 3 W2 — file-write persistence ingestion + SENTRY-008. Today Sentry watches
credential READS only; the entire persistence-injection attack class (writes to
~/.claude/settings.json, .vscode/tasks.json, LaunchAgents, systemd-user units) is
invisible (analysis G-B: A2/D1/D2/E1 all NONE). This plan adds a distinct
`EventFileWrite` event kind, the SENTRY-008 persistence-write rule, and the per-OS
write ingestion that fuels it — including fixing a real macOS new-file create-parse
bug and the mislabeled Windows Kernel-File event IDs.

Purpose: persistence is the whole attack for these patterns; the rule (SENTRY-008)
is cheap but its FUEL is the per-OS write ingestion this plan lands.

Output: EventFileWrite kind; persistenceWritePaths + isPersistencePath + evalSENTRY008;
Linux 2nd fanotify group (>=5.9); macOS write/rename + new-file union fix; Windows
correct write IDs.
</objective>
