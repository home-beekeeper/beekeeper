<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Go module proxy -> beekeeper CI `go install` | The pinned module spec crosses from the proxy into CI; a typosquat or unpinned spec could install a wrong binary |
| pollen binary (PATH) -> beekeeper scan subprocess | beekeeper invokes whatever `pollen` is on PATH; the version is the trust parameter |
| maintainer keyboard -> GitHub (push/tag/cosign) | Irreversible, auth-gated outward actions; centralized in the runbook for controlled execution |
