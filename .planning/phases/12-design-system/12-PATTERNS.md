### `web/lib/reduced-motion.tsx` (hook + provider, event-driven ŌĆö CREATE NEW)

**Analog:** NEW ŌĆö no in-repo analog exists.

**Pattern source:** UI-SPEC ┬¦ReducedMotionProvider Contract (lines 711-752); this is the
verbatim implementation contract ŌĆö copy it exactly.

```tsx
/* web/lib/reduced-motion.tsx */
"use client";
import { createContext, useContext, useEffect, useState } from "react";

interface ReducedMotionContextValue {
  prefersReducedMotion: boolean;
}

const ReducedMotionContext = createContext<ReducedMotionContextValue>({
  prefersReducedMotion: false,
});

export function ReducedMotionProvider({ children }: { children: React.ReactNode }) {
  const [prefersReducedMotion, setPrefersReducedMotion] = useState(false);

  useEffect(() => {
    const mq = window.matchMedia("(prefers-reduced-motion: reduce)");
    setPrefersReducedMotion(mq.matches);
    document.documentElement.dataset.reducedMotion = mq.matches ? "true" : "false";

    const handler = (e: MediaQueryListEvent) => {
      setPrefersReducedMotion(e.matches);
      document.documentElement.dataset.reducedMotion = e.matches ? "true" : "false";
    };
    mq.addEventListener("change", handler);
    return () => mq.removeEventListener("change", handler);
  }, []);

  return (
    <ReducedMotionContext.Provider value={{ prefersReducedMotion }}>
      {children}
    </ReducedMotionContext.Provider>
  );
}

export function useReducedMotion(): boolean {
  return useContext(ReducedMotionContext).prefersReducedMotion;
}
```

**Phase 16 contract** (do not change this interface): Phase 16 `Hero3D.tsx` calls
`useReducedMotion()` and conditionally skips mounting the R3F canvas. The hook signature
`(): boolean` is locked.

**`data-reduced-motion` attribute:** The `document.documentElement.dataset.reducedMotion`
write syncs the CSS `[data-reduced-motion="true"]` selector alongside the React context,
giving CSS-only consumers (non-React contexts, raw `<style>` tags) access to motion state.

## Shared Patterns

### Pattern: Tailwind `@theme inline` with indirect var references

**Source:** RESEARCH.md ┬¦Pattern 1 (lines 186-202)
**Apply to:** `web/app/globals.css`

When `@theme` values are themselves `var(ŌĆ”)` references, `@theme inline` is MANDATORY.
`@theme` (non-inline) would emit `color: var(--color-background)` ŌĆö a reference to a var
that is itself a var, which Tailwind v4 cannot resolve for utility overrides.

```css
/* CORRECT ŌĆö inline resolves the chain */
@theme inline {
  --color-background: var(--color-bk-bg);
  --font-sans: var(--font-inter);
}

/* WRONG ŌĆö do not use for indirect refs */
@theme {
  --color-background: var(--color-bk-bg);
}
```

### Pattern: `@custom-variant dark` syntax

**Source:** RESEARCH.md ┬¦Pattern 2 (line 229) + Anti-Patterns (line 328)
**Apply to:** `web/app/globals.css`

```css
/* CORRECT ŌĆö matches elements that ARE .dark AND elements inside .dark */
@custom-variant dark (&:where(.dark, .dark *));

/* WRONG ŌĆö misses elements with .dark directly on them */
@custom-variant dark (&:is(.dark *));
```

### Pattern: `cn()` import in shadcn components

**Source:** RESEARCH.md ┬¦cn() utility (lines 527-537)

## Anti-Patterns (Planner Must Gate Against)

These are explicitly listed in RESEARCH.md ┬¦Anti-Patterns (lines 325-332) and ┬¦Pitfalls:

| Anti-Pattern | How It Manifests | Gate |
|-------------|-----------------|------|
| Leave shadcn-generated globals.css as-is | OKLCH zinc colors render instead of Beekeeper tokens | Immediately overwrite after `shadcn init` |
| `@theme` without `inline` for var references | Tailwind utilities emit unresolved var chains | Verify `@theme inline` (not `@theme`) in globals.css |
| Wrong dark variant: `(&:is(.dark *))` | Elements with `.dark` directly miss dark: utilities | Use `(&:where(.dark, .dark *))` |
| `shadcn init --monorepo` from repo root | Creates wrong directory structure | Always run `shadcn init` from inside `web/` without `--monorepo` |
| Geist className left on `<html>` | Font variable conflict | Phase 12 replaces entire layout.tsx ŌĆö not a partial edit |
| `suppressHydrationWarning` absent | React hydration mismatch; possible theme flicker | Required on `<html>` element ŌĆö not optional |
| Fumadocs imports uncommented before Phase 13 | Build fails (package not installed) | Keep as comments; Phase 13 uncomments after installing packages |
