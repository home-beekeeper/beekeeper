<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args / stdin → policy commands | A sample tool-call JSON and a policy file path cross here; both are untrusted input parsed before evaluation |
| beekeeper-self feed → enforcement startup | The self-catalog result crosses into the refuse-to-run decision on every enforcement command |
| running binary → audit log + stderr | Self-quarantine evidence crosses here and must be recorded before exit |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-18 | Tampering | policy validate/test parsing an adversarial file | mitigate | Reuses policyloader.LoadPolicyFile + ValidateSchema (Plan 01) which reject url/exec/unknown fields before any evaluation (V5/V10) |
| T-09-19 | Spoofing/Tampering | compromised binary skips the self-quarantine guard | mitigate | enforceSelfQuarantine is called at the top of every enforcement RunE; CheckSelfCatalog verifies a separately-keyed signed feed (Plan 03); a fail-closed result refuses to run |
| T-09-20 | Repudiation | self-quarantine fires without evidence | mitigate | Guard writes a self_quarantine audit record AND a stderr warning BEFORE returning the non-zero error (V7) |
| T-09-21 | Denial of Service | self-quarantine guard bricks diagnostics, leaving the user no recovery path | mitigate | Guard runs only on enforcement commands; version/diag/selftest/policy-validate stay runnable so the developer can diagnose and follow the verification path (Open Question 3) |
| T-09-22 | Elevation of Privilege | project config silently relaxing fail mode for a malicious repo | accept | By design (PRD §9); documented in config comments (Plan 02) and in THREAT-MODEL.md section 6; developer controls trusted repos |
| T-09-23 | Information Disclosure | diag output leaking secrets | mitigate | diag prints only latencies, counts, content hashes, and degraded flags (Plan 04) — no tokens or secret-bearing paths |
</threat_model>
