<objective>
Implement the `beekeeper-self` self-quarantine catalog client (CTLG-04, SFDF-06) — the safety-critical core of the capstone. A static, signed JSON feed lists known-compromised Beekeeper versions; Beekeeper fetches it, verifies its signature against a SEPARATE embedded public key, and self-quarantines if the running `version.Version` matches. The non-trivial requirement is the fail-closed-vs-network tension: an INVALID SIGNATURE fails closed (proven integrity failure), but a NETWORK failure degrades to read-and-notify (warn + continue) so a flaky connection or fresh install does not brick the tool. Self-quarantine state persists in state.json and is honored offline on subsequent runs.

Purpose: Close the self-compromise detection loop (Phase 9 success criterion 4). Compromising both the release pipeline and the self-catalog requires defeating two independent security postures.
Output: New `internal/catalog/selfcatalog.go` + `selfkey.go`, a `SelfQuarantineState` extension to `WatchState`, the self-catalog `MultiCatalogLookup` adapter, tests, and three feed fixtures. This is Wave 0 for the CLI startup wiring in Plan 05.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| beekeeper-self host/CDN → CheckSelfCatalog | The feed host is a network-attacker target; a fake or unsigned feed crosses here and must be rejected before applying |
| network layer → CheckSelfCatalog | Transient DNS/timeout failures cross here and must NOT be treated as compromise |
| state.json → startup check | A persisted self-quarantine marker crosses here on every run and must hold offline |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-09 | Spoofing/Tampering | fake beekeeper-self feed at attacker URL | mitigate | Ed25519 signature verified against a SEPARATE hardcoded public key embedded at compile time (selfkey.go); a forged feed fails verification (V6) |
| T-09-10 | Tampering | compromised CDN serves unsigned/tampered feed | mitigate | Signature REQUIRED; invalid signature returns errIntegrity → fail-closed, never warn (V6/V7) |
| T-09-11 | Denial of Service | transient network failure bricks the tool | mitigate | errNetwork is distinct from errIntegrity; no-cache or stale-cache → warn+continue, never exit (Pitfall 2) |
| T-09-12 | Elevation of Privilege | release-pipeline key reuse forges self-catalog | mitigate | Self-catalog key is independent of the release-signing identity; defeating both requires two compromises (SFDF-06) |
| T-09-13 | Repudiation | self-quarantine fires without an audit trail | mitigate | Self-quarantine writes SelfQuarantineState (version, entry_id, reason, fired_at) to state.json atomically; Plan 05 also logs to audit + stderr before exit (V7) |
</threat_model>
