<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| agent tool call → check adapter | Untrusted file_path / path / Bash command strings are extracted and canonicalized here before reaching the pure matcher |
| process env → canonicalization | os.UserHomeDir / os.Getenv(%USERPROFILE%) read here; values feed matching only — the command string is NEVER executed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-05 | Information Disclosure | Path traversal (`../../.aws/credentials`) | mitigate | canonicalizePath runs filepath.Abs before matching (SPATH-02); covered by TestCanonicalizePath traversal case |
| T-07-06 | Information Disclosure | Tilde bypass (`~/.aws/credentials` left literal) | mitigate | expandHome (os.UserHomeDir + filepath.Join), copied from watch prior art; fail-safe returns input on home-resolve error |
| T-07-07 | Information Disclosure | Symlink-to-credential dereference / missing-file allow | mitigate | filepath.EvalSymlinks with fallback to Abs (Pitfall 3) — non-existent credential paths still evaluate, do not silently allow |
| T-07-08 | Information Disclosure | Shell-command wrapping (`cat ~/.ssh/id_rsa`, `type %USERPROFILE%\.ssh\id_rsa`) | mitigate | extractBashCredentialPaths verb-prefix scan + canonicalizePath %USERPROFILE% expansion (SPATH-03, D-01) |
| T-07-09 | Spoofing | Windows backslash bypass (`C:\Users\u\.aws\credentials`) | mitigate | filepath.ToSlash normalization; pure matchesBlockPattern also normalizes — defense in depth |
| T-07-10 | Elevation of Privilege | Executing the extracted command during "expansion" | accept→mitigate | Expansion is string-replace only (os.Getenv); the Bash command is never passed to a shell — documented in code comment (D-01) |

No `high`-severity unmitigated threats. T-07-10 risk is eliminated by design (no execution path exists).
