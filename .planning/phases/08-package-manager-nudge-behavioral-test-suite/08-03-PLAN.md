<objective>
Build the PURE heart of the nudge feature in a new `internal/nudge/` package: the decision
function `Evaluate(cmd pkgparse.ParsedCommand, state PMState, cfg Config) Decision`, the closed
reason-code enum, the config + defaults (with Flag 5 corrections baked in), the pure command
rewriter, and the pure version/floor compare. This mirrors the locked `policy.EvaluateReleaseAge`
pattern exactly: a pure decision over a caller-resolved input struct. ALL detection I/O is deferred
to Plan 04's `detect.go` adapter — this plan touches NO I/O. The table-driven test suite here
satisfies PRD §10 acceptance criteria 1-10 (BTEST-01).

This plan also defines the closed §9 `nudge_action` enum (`advise|proceed|rewrite|block`) via an
`ActionString(Action) string` mapper, so the audit record (Plan 05 field, Plan 06 wiring) can carry
the PRD §9 `decision` semantics WITHOUT colliding with the repo's existing `allow|warn|block`
audit `Decision` field. The two are deliberately separate: `Level` drives mergeDecisions/exit code;
`nudge_action` preserves PRD §9 wording for forensic conformance (resolves BLOCKER 2 / §10-14 root cause).

Purpose: This is the CLAUDE.md purity discipline applied to nudge — `internal/policy` is the pure
library; nudge gets its own pure decision in its own package (own Action/PMState/reason enum) so the
locked library stays lean, enforced by `TestNudgeEvaluateImportsArePure`.
Output: `internal/nudge/{evaluate,config,reasons,rewrite,version}.go` + their tests — pure, table-driven.
</objective>
