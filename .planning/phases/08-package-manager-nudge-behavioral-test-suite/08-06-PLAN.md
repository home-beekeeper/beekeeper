<objective>
Wire the nudge into the three live enforcement consumers and add the gateway drift check:

1. **check hook** (`runCheck` + new `nudge_adapter.go`) — parse the Bash command via
   `pkgparse.Parse`; when `IsInstall`, resolve `PMState` FRESH via `nudge.DetectState` (NO cache —
   Flag 2 Position B, the one-shot hook gets no cache hits), call the pure `nudge.Evaluate`, map
   the nudge `Decision` to a `policy.Decision`, and `mergeDecisions` it AFTER `ApplyPolicyOverlay`
   (CR-02 ordering — an allow overlay must not downgrade a nudge Block). Emit a `record_type:"nudge"`
   audit record carrying the §9 fields AND `nudge_action` (= nudge.ActionString).
2. **gateway** (`policy.go` + `proxy.go`) — same merge, but this is the ONLY place the 60s
   `nudge.Cache` is constructed (once at start) and reused per request. The gateway's self-contained
   `gateway.Config` does NOT embed `config.Config`, so it has no nudge field today (WARNING 3): this
   plan ADDS a `Nudge nudge.Config` field to `gateway.Config` and has `newGatewayHandler` DEFAULT a
   zero-value `cfg.Nudge` to `nudge.DefaultConfig()` (so the gateway is safe even before the daemon
   literal sets it). `applyPolicy` then reads `cfg.Nudge` (it now exists). The DAEMON-side population
   of `gatewayCfg.Nudge = nudge.ConfigFrom(...)` from the layered `cfg.Nudge` lives in
   `cmd/beekeeper/main.go`'s `newGatewayCmd` — that file is owned by Plan 08 (which already owns
   `cmd/beekeeper/main.go` for the nudge/config command registration), so this plan does NOT touch
   `cmd/beekeeper/*` and the two Wave-4 plans keep ZERO `files_modified` overlap. The at-most-one-
   advisory-per-session cap (NUDGE-03, RESEARCH Pitfall 4 / Open Q3) is gateway-session state keyed by
   agent-id when present, else process-global.
3. **shim** (`shim.go`) — the npm shim calls `pkgparse.Parse` → `nudge.Evaluate` before proxying.
4. **gateway drift check** (`drift.go`, §10-15 / Open Q2) — a periodic (weekly, per
   cfg.Nudge.MajorDriftCheck.Interval) task in the long-lived gateway with an INJECTED metadata-
   fetch fn (returning e.g. "pnpm 12.0.0"); it calls the EXPORTED `nudge.IsMajorDrift` (Plan 03 —
   the lowercase isMajorDrift is uncallable from package gateway), and when a new major is found
   emits an ASYNC, non-blocking `record_type:"version_drift"` severity-info record. It NEVER runs on
   the check hot path and does NOT auto-update floors (PRD §7.1, Out-of-Scope).

Purpose: This is the Phase 7 most-restrictive-merge pattern repeated for nudge. Soft mode
(Advise/Rewrite → Level "warn") keeps exit 0; only `Block` (requireHardened) is restrictive — so
the merge keeps `exitCodeFor` semantics intact.
Output: live nudge wiring in check/gateway/shim + the gateway drift check + the gateway.Config.Nudge
carrier field (with a safe DefaultConfig fallback). The daemon literal that fills it from layered
config is Plan 08. (Behavioral tests proving the wiring are Plan 07.)
</objective>
