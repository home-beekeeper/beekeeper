none of which depend on the nudge decision types — so this runs in Wave 1 in parallel with Plan 01:

   mirroring the Phase 5/6 field-addition pattern.

   struct (config imports only stdlib — no import of internal/nudge, avoiding a cycle); a missing
   block resolves to documented defaults; project `.beekeeper.json` `nudge.enabled:false` wins.

Purpose: Decouple the E2E-enabling env override, the audit schema extension, and the config block
from the nudge decision core so they can be built first and in parallel — this removes the A2
blocker that the live-binary release gate hangs on.
