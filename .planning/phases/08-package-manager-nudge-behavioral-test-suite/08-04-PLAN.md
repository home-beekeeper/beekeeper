<objective>
Build the IMPURE detection adapter for the nudge feature: `detect.go` (the only I/O in the
feature) shells out to `pnpm/bun/node --version` under a 2s hard timeout, scans `bunfig.toml` /
`pnpm-workspace.yaml`, and produces the `PMState` that Plan 03's pure `Evaluate` consumes. It also
provides the gateway-only 60s `Cache` (Flag 2 Position B — the cache lives ONLY in the long-lived
gateway, NEVER in the one-shot check hook). Hand-written config scanners (no TOML/YAML dependency
per REQUIREMENTS Out-of-Scope) carry never-panic fuzz targets (BTEST-03 release gate).

Purpose: This is the impure-adapter half of the locked pure-decision-over-resolved-input pattern,
mirroring `internal/check/paths.go` feeding `policy.EvaluatePath`. Detection failure is fail-OPEN
by design (a slow/absent PM must not block the agent — PRD §10-12); this is the documented
soft-nudge exception to the catalog/path fail-closed rule.
Output: `internal/nudge/{detect,scanners}.go` + `{detect,scanners}_test.go` + `scanners_fuzz_test.go`.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| detect.go → pnpm/bun/node binaries (PATH) | The adapter execs external binaries resolved via PATH |
| project root / ~ → scanners | Attacker-influenced bunfig.toml / pnpm-workspace.yaml contents are read and parsed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-10 | Elevation (PATH hijack / arbitrary exec) | detect.go exec | mitigate | Detection execs a FIXED argv ("pnpm"/"bun"/"node", "--version") via exec.CommandContext — no user-controlled path or argument is ever passed to the exec call. Bounded by a 2s timeout. Never execs file contents. |
| T-08-11 | Denial of Service (slow PM blocks check budget) | detect.go timeout | mitigate | 2s hard timeout per exec via context.WithTimeout; on timeout the PM is treated as not installed and the agent proceeds (fail-OPEN by design, §10-12). The gateway Cache amortizes the cost (one slow call per 60s, not per request — Pitfall 1). |
| T-08-12 | Denial of Service (malformed config crashes the security tool) | scanners.go | mitigate | scanBunfig/scanPnpmWorkspace return safe defaults on parse trouble and NEVER panic; FuzzBunfig + FuzzPnpmWorkspace are release-gate fuzz targets proving this (§10-13, BTEST-03). |
| T-08-13 | Information disclosure (reading arbitrary files) | scanners.go file read | mitigate | Scanners read ONLY bunfig.toml (project root + ~/.bunfig.toml) and pnpm-workspace.yaml (project root) — fixed, bounded paths; read-only; contents are parsed, never executed (V12). |
</threat_model>
