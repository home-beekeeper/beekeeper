## CANONICAL CONTENT (single source of truth — use verbatim in BOTH repos)

### C1 — Install-posture concept paragraph
Beekeeper enforces a default install posture at the agent pre-exec hook. Three rules evaluate every
package install an agent runs: release age (a package first published less than 24 hours ago warns),
lifecycle scripts (an install that would run pre or post-install scripts warns), and git or remote-URL
dependencies (a dependency pulled from a git ref or arbitrary URL instead of a registry warns). All
three warn by default and fail soft: an unknown answer (a registry timeout, a missing publish
timestamp) warns rather than blocks, so a slow or offline check never stops an agent. It is tool
agnostic: the same checks apply at the hook regardless of which package manager the agent reaches for.

### C2 — npm v12 obsolescence note (home + getting-started)
npm v12 blocks install lifecycle scripts by default, so the old steer-to-pnpm-or-Bun nudge is no longer
the headline win. Install posture is the tool-agnostic successor: it applies the same structural checks
to every package manager, at the hook.

### C3 — Boundary statement (VERBATIM from internal/posture/boundary.go; long form for security/configuration, short elsewhere)
LONG (`posture.BoundaryStatement`):
> Install posture is enforced pre-execution at the agent hook for hooked harnesses that support it,
> inheriting each harness tier's caveats. For harnesses with no pre-exec hook, and for installs a person
> runs directly in a terminal, it is observed and audited by the Sentry layer rather than prevented,
> unless the experimental package-manager shim is installed. The MCP gateway is not a general install
> surface. The shim extends pre-exec enforcement to installs run through the shimmed PATH, but it is
> limited: it can be bypassed by calling a tool by its absolute path, and it requires adding the shim
> directory to your PATH. It is not a headline v1.0 guarantee.

SHORT (`posture.BoundaryShort`):
> Posture is prevented at the hook for hooked harnesses, and experimentally at the PATH shim for
> human-run installs; everything else Sentry observes and audits without preventing.

### C4 — SENTRY-009 line (add to the SENTRY-001..008 enumerations)
SENTRY-009: install observation. The Sentry layer observes and audits package-manager installs by a
monitored descendant, including installs a person runs directly in a terminal and installs on harnesses
with no pre-exec hook. It is detection-only process attribution (the package-manager PID/PPID, exe, and
cmdline); it adds NO registry I/O and does NOT evaluate posture in the privileged daemon (posture
matching is the hook's job). It records (`sentry_install_observed`, severity info) but never blocks,
kills, or quarantines.

### C5 — `beekeeper posture` CLI surface (replaces the `beekeeper nudge` doc section)
```
beekeeper posture            Show each package manager's install posture side-by-side with
                             Beekeeper's enforced posture (read-only, machine-wide). Names the gaps.
beekeeper posture --full     Print the full enforcement-boundary statement.

beekeeper posture allow <package>   Grant a scoped, audited exception to the install posture.
    --once               Allow the next matching install, then warn again.
    --always             Record a standing exception (requires --reason).
    --rule <rule>        Scope to one rule: release-age | lifecycle | git-remote.
    --ecosystem <eco>    Scope to one ecosystem (e.g. npm); empty matches any.
    --reason "<text>"    Recorded justification (required for --always).
  Posture-scoped: silences a posture warn for the package but NEVER downgrades a catalog malware
  block. Each invocation writes a distinct posture_override audit record.

beekeeper posture enforce <rule>    Set a posture rule's action.
    <rule>     One of: release-age | lifecycle | git-remote
    --block    Block a definite violation of the rule (opt it up from warn).
    --warn     Lower the rule back to the default warn.
  The unknown / fail-soft path always warns: a registry outage never blocks an install even under
  --block. Writes a distinct posture_override audit record.
```
Posture override audit `posture_override_action` values: `allow_once`, `allow_always`, `enforce_block`, `enforce_warn`.
