<objective>
Close verification gap 4 (SC-5 / TUI-09, and the TUI-06 drill-down half). `PolicyPanel` is a static stub: `Update` is a no-op, the body is hardcoded prototype strings, the e/t footer keys are labeled "visual-only in Phase 8", and there is no rule list with enabled/disabled state. The success criterion requires that with `--admin`, the developer can toggle individual policy rules directly from the TUI (the quarantine half of SC-5 is already done).

KEY CONSTRAINT discovered during planning: there is no declarative JSON policy-file format yet — CODE-01 (declarative `policies/` files loaded by the engine) is a Phase 9 deliverable, and `internal/policy` exposes only pure evaluation with hardcoded default thresholds, no on-disk rule file. To close this gap WITHOUT pulling Phase 9's full policy-as-code forward, this plan defines a minimal, self-contained policy-rules file under `~/.beekeeper/policies/` that the TUI owns: a small JSON document listing the named rules with an `enabled` boolean each. The panel loads it (seeding a default file from the locked prototype rule names on first run), renders each rule as a drill-down row with its enabled/disabled state, and — gated on admin mode — toggles and persists the `enabled` flag. Phase 9's richer policy-as-code can supersede this file format; the TUI reads whatever rules the file contains.

This plan:
1. Adds `internal/tui/policy_rules.go`: a `PolicyRule{ID, Label, Detail string; Enabled bool}` type, `LoadPolicyRules(policiesDir)` (reads `<policiesDir>/tui_rules.json`; seeds it with the locked prototype rules — corroboration, release-age, lifecycle, sentry baseline, llamafirewall — all enabled, on first run), and `ToggleRule(policiesDir, id string, enabled bool)` (persists the flag, 0600).
2. Rewrites `policy_panel.go` to hold `rules []PolicyRule`, `selIdx int`, `adminMode bool`, and a `policiesDir`; constructor `NewPolicyPanel(adminMode bool)` calls LoadPolicyRules; Body renders each rule row with an ON/OFF state; Update handles j/k navigation always and e/t toggle ONLY when adminMode (mirroring QuarantinePanel's r/p admin gate), calling ToggleRule and reloading.
3. Wires model.go `runPaletteSelection` to construct `NewPolicyPanel(a.adminMode)` (it currently calls `NewPolicyPanel()` with no arg).

Wave 2 — depends on 08-08 because BOTH modify model.go (serialized to avoid same-wave file conflict). All other files (policy_panel.go, policy_rules.go, policy_panel_test.go) are disjoint from every other gap plan.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| tui_rules.json on disk → PolicyPanel | The rules file is user-owned config; a malformed/older file is untrusted and must not crash the dashboard |
| e/t key → ToggleRule write | Only an admin-mode panel may write; the write changes a local 0600 file, not live enforcement |
| adminMode flag → toggle gate | adminMode originates from the `--admin` CLI flag threaded App → panel; it is the single authorization signal for write actions |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-07-01 | Elevation of Privilege | A non-admin session toggles a policy rule | mitigate | Update gates e/t behind `if !p.adminMode { handle only navigation; return }`, mirroring the verified QuarantinePanel r/p gate; NewPolicyPanel(a.adminMode) is the only way adminMode is set |
| T-08-07-02 | Tampering | Malformed/hand-edited tui_rules.json crashes the panel or the dashboard | mitigate | LoadPolicyRules is fail-soft: any read/unmarshal error returns seeded defaults; no panic on bad data |
| T-08-07-03 | Tampering | A toggle write races a concurrent external edit to tui_rules.json | accept | Single-user local dashboard; last-write-wins on a 0600 file is acceptable for v1; Phase 9 policy-as-code owns the authoritative format |
| T-08-07-04 | Information Disclosure | Rules file world-readable exposes policy posture | mitigate | File written 0600 and dir 0700, mirroring config.Save; owner-only |
| T-08-07-05 | Repudiation | Toggling a rule in the TUI is not audited | accept | This gap closes the TUI toggle affordance; audit-trailing TUI admin actions is a Phase 9 policy-as-code concern, out of scope for gap closure |
</threat_model>
