<objective>
Close verification gaps 1 and 2 (TUI-02, TUI-03) in the live activity feed / Sentry alerts panel. Today `AlertsPanel.recordToRow` silently drops `allow` decisions, never reads `AgentName`, and the footer advertises "enter inspect" but the `enter` key has no handler — file/network detail is collapsed into a one-line meta string and the process tree is never read.

This plan:
1. Adds an `Agent` field to `AlertRow` populated from `rec.AgentName`, plus an agent-identity column in the row render (TUI-02).
2. Stops filtering out `allow` policy decisions — they appear with `BadgeOK()` so the feed shows the full allow/warn/block spectrum (TUI-02).
3. Adds expand state (`expanded bool`) and an `enter`-key handler that toggles a detail view for the selected row, showing the stored `SentryParentChain`, full `SentryFilesAccessed`, and full `SentryNetworkDests` (TUI-03).

Wave 1 — self-contained; touches only `alerts_panel.go` and its test. No dependency on other gap plans (disjoint files from 08-07/08-08/08-09).
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| audit NDJSON record → AlertRow render | AuditRecord fields (AgentName, file paths, network dests) are attacker-influenceable strings written by upstream tool calls; the TUI renders them, never evaluates them |
| selected row → expanded detail | Expanded view reads only slices already stored on the row; no new I/O, no re-parse of the audit file |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-06-01 | Tampering | Malicious AgentName / file path / network dest in audit record renders into the TUI | mitigate | Render as plain styled text only; truncate/pad to fixed column width; never exec, eval, or interpret the strings. Lipgloss escapes control via terminal styling, not shell |
| T-08-06-02 | Denial of Service | Very large SentryFilesAccessed / SentryNetworkDests slice in a crafted record floods the expanded view | mitigate | Detail block is bounded by panel height; rows are already capped at maxAlertRows=200; expanded view renders within the provided width/height and truncates long paths |
| T-08-06-03 | Information Disclosure | Expanded detail surfaces full credential file paths read by an attacker | accept | This is the intended forensic signal (TUI-03) — the operator must see exactly which credential files were touched to respond; data is local audit only, never transmitted |
| T-08-06-04 | Denial of Service | nil ParentChain/Files/Network on a policy_decision row when expanded | mitigate | Each detail loop guards an empty/nil slice and renders "(none)"; no nil dereference |
</threat_model>
