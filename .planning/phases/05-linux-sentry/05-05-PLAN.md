    Startup sequence (exact order per CONTEXT.md §SLNX-10 and daemon startup pattern):
    ```
    1. rlimit.RemoveMemlock()
    2. tier := ProbeTier()
    3. if tier <= Tier1: load eBPF objects (loadBeekeeperExecObjects + loadBeekeeperNetObjects)
    4. DropCapabilities(keepCaps(tier))
    5. ApplySeccomp()
    6. fanFd, _ := InitFanotify(tier); FanotifyMarkPaths(fanFd, cfg.Policy.SensitivePaths or defaultSensitivePaths)

    8. sdNotifyReady()
    9. Start goroutines: events channel (cap 10000), start goroutines, start IPC server
    10. <-ctx.Done()
    11. Cleanup: close links, close fanotify fd, close IPC server, drain events channel
    ```

    Goroutines started (in order after sdNotifyReady):

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| systemd → beekeeper sentry process | Daemon runs as root; unit file must not be world-writable; capability bounding set enforced by systemd |
| IPC handler → daemon state | Handler runs in a goroutine; ruleStates map must be mutex-protected |
| CLI → IPC socket | CLI verifies socket path is in state dir (under ~/.beekeeper/); no path traversal |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-05-01 | Elevation of Privilege | Unit file written to /etc/systemd/system/ by non-root | mitigate | protect install checks os.Getuid()==0 first; returns clear error if not root |
| T-05-05-02 | Tampering | IPC ruleStates map race between correlationEngineLoop and ipcHandler goroutine | mitigate | sync.RWMutex protects ruleStates; correlationEngineLoop holds read lock during EvaluateEvent; ipcHandler holds write lock during enable/disable |
| T-05-05-03 | Information Disclosure | IPC StatusResponse leaking process tree contents or event payloads | mitigate | StatusResponse contains only aggregate counts (EventsProcessed, EventsDropped, RulesActive); no individual event data |
| T-05-05-04 | Denial of Service | Daemon failing to send sdNotifyReady causing systemd to kill and restart it in a loop | mitigate | sdNotifyReady is called after all goroutines are started; NOTIFY_SOCKET absent = no-op (foreground run); systemd RestartSec=5s limits restart rate |
| T-05-05-05 | Tampering | Audit records for baseline events written as sentry_alert (enforcement) accidentally triggering quarantine | mitigate | alertToAuditRecord checks alert.BaselineMode and sets RecordType to sentry_alert_baseline; Decision set to "warn" not "block" during baseline |
</threat_model>
