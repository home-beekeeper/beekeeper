# Plan 26-02 — Remove the nudge, relocate the pm-config readers, repoint the shim

## Goal

Remove the package-manager nudge feature and its steer-to-pnpm/Bun behavior everywhere, while PRESERVING (relocating) the two pieces worth keeping: the package-manager detection (`detect.go`) and the pm-config scanners (`scanners.go`, which read `.npmrc`/`pnpm-workspace.yaml` `minimumReleaseAge` + `bunfig.toml`). Those move into a new `internal/posture` package to power the Layer-2 `beekeeper posture` view (Phase 28). The release-age policy logic is already preserved in `internal/policy/release_age.go` (it is separate from the nudge); this plan does not touch it.

After this plan: `internal/nudge` no longer exists; nothing steers npm/yarn to pnpm/bun; the hook no longer runs a nudge; the shim routes through `beekeeper check`; the build/vet/test are green.

## Hard constraints (do not violate)

1. **Frozen corpus schema.** `audit.AuditRecord` is embedded in the frozen `CorpusRecord` (CorpusSchemaVersion 1.0). DO NOT remove the `NudgeAction` / `PMState` fields from `AuditRecord` — that would break the frozen schema. Instead: stop POPULATING them (remove the nudge record writers), and add a `// Deprecated:` comment. Keep `redact.go`'s handling of those fields (harmless). Leave the `"nudge"` record_type string handling tolerant; do not assert it anywhere new.
2. **Purity preserved.** `internal/pkgparse` and `internal/policy` stay import-pure (their purity tests must still pass).
3. **No behavior wired to the hook here.** Removing the handler's nudge block leaves install posture temporarily unenforced at the hook — that is fine; Phase 27 wires posture in. The milestone does not ship between phases.
4. **Shim is kept, not deleted** (maintainer decision). Repoint it; document it as roadmap/experimental.

## Tasks

### Task 1 — Relocate the pm-config readers into `internal/posture`
- Create `internal/posture/` package.
- Move `internal/nudge/detect.go` and `internal/nudge/scanners.go` (and ONLY the helpers they compile-depend on, e.g. parts of `version.go` if referenced) into `internal/posture`, renaming `package nudge` → `package posture` and any exported identifiers that read awkwardly (keep behavior identical). These detect installed pm versions and read each pm's config (`minimumReleaseAge`, bun scanner, hardening state).
- Move their tests (`detect_test.go`, `scanners_test.go`, `scanners_fuzz_test.go`, and the relevant parts of `version_test.go`) into `internal/posture`, renaming the package. The fuzz scanners gate must still build (`-tags fuzz`).
- Drop anything in those files that only served the steering decision (e.g. "preferred manager" knobs) if not needed for read-only detection.

### Task 2 — Delete the nudge steering
Delete these files entirely:
- `internal/nudge/evaluate.go` + `evaluate_test.go` + `evaluate_bun_test.go`
- `internal/nudge/rewrite.go` + `rewrite_test.go`
- `internal/nudge/reasons.go` + `reasons_test.go`
- `internal/nudge/config.go` + `config_test.go` (the `nudge.Config`/`DefaultConfig`/`ConfigFrom` steering config)
- whatever remains of `internal/nudge/version.go` once detection's deps are relocated (the `IsMajorDrift`/floor logic was nudge-only)
- `internal/check/nudge_adapter.go` + `nudge_adapter_test.go`
- `cmd/beekeeper/nudge.go` + `nudge_test.go` (the `beekeeper nudge` subcommand)

End state: the `internal/nudge` directory is gone.

### Task 3 — Remove the wiring
- `internal/check/handler.go`: remove the nudge block (~lines 361-385, the `evaluateNudge(...)` call + `nudgeCfgValue`). Remove the now-unused `config.DefaultNudgeConfig` reference.
- `cmd/beekeeper/main.go`: remove the `nudge` import, `newNudgeCmd()` registration, and `ensureNudgeBlockDefault` (function + its call in `hooks install`). The "Enabled supply-chain enforcement: nudge.mode=block" copy is deleted (NMIG-01 steer-copy removal).
- `cmd/beekeeper/config.go`: remove the `config set nudge.*` keys/branches (and `config_test.go` cases). If `config set` becomes empty of keys, keep the command but with no nudge keys (or whatever non-nudge keys exist).
- `internal/gateway/policy.go`: remove `nudgeDecisionFor`. `internal/gateway/proxy.go`: remove its call site + any cached PMState plumbing for nudge. `internal/gateway/drift.go` + `drift_test.go`: the version-drift check was a nudge feature — remove it (and its gateway wiring). Update `gateway_test.go` to drop nudge-merge/drift assertions.
- `internal/config/config.go`: remove `NudgeConfig`, `DefaultNudgeConfig`, and the `Nudge` field on `Config`. `internal/config/layered.go`: remove `mergeNudge` and the nudge-specific TMD invariants (untrusted-can't-disable-nudge). `internal/config/accessors.go`: remove nudge accessors. Update `config_test.go`, `layered_test.go`, `accessors_test.go` to drop nudge cases. (The TMD intent — untrusted layers cannot relax security — is re-expressed for posture in a later phase if posture gains config; do not invent posture config now.)
- `internal/audit/types.go`: keep `NudgeAction`/`PMState` fields (frozen schema) but add `// Deprecated: nudge removed in v1.1.0; field retained for corpus schema compatibility, no longer populated.` Do not remove them. `redact.go`: leave as-is (harmless).
- `internal/tui/audit_panel.go`: remove nudge-specific display rows/labels; keep the panel working for the remaining record types. Update `audit_panel_test.go`.
- `internal/coveragegate/floor.go`: replace the `internal/nudge` per-package floor entry with `internal/posture` (or remove the nudge entry). Keep the gate green.

### Task 4 — Repoint the shim (keep + document as roadmap)
- `internal/shim/shim.go`: remove `NudgeCheck` + `NudgeResult` (the nudge-before-proxy helper) and the `internal/nudge` import. The generated shim scripts already invoke `beekeeper check --tool <tool> --args …`, which routes through the posture-aware hook once Phase 27 lands — so the shim inherits posture with no per-pm logic. Update `shim_test.go` to drop the nudge-before-proxy assertions; keep install/uninstall/status/real-binary tests.
- Update the `internal/shim/shim.go` package doc to state the honest boundary: the shim is an experimental/roadmap surface for machine-wide human-run installs; it is not a headline v1.0 install-posture surface, and an install invoked by absolute path bypasses it.
