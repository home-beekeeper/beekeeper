# Plan 26-01 — Git / remote-URL install detection

## Goal

Add the third install-posture rule that does not exist yet: detect installs that pull from a git ref, a remote URL, a tarball URL, a `github:`/`gitlab:` shorthand, or a local `file:` spec, and flag them. This is a NEW pure rule. It is the "git and remote URL dependencies" piece of the PRD Layer 1 default posture. Default action is WARN (PRD: "flagged and warned on first encounter, not hard-blocked, with the reason surfaced").

This plan adds ONLY the pure detection + evaluation + tests. Wiring into the hook is Phase 27.

## Context (verified)

- `internal/pkgparse/pkgparse.go` parses install commands but has NO remote-source field. `ParsedCommand.Package`/`Version` are derived by `splitVersion` on the last `@`, which mis-handles a `git+https://…` or URL token.
- The existing pure posture evaluators to mirror exactly: `internal/policy/release_age.go` (`EvaluateReleaseAge`) and `internal/policy/lifecycle.go` (`EvaluateLifecycle`) — pure, imports only `fmt`/`strings`, return a `policy.Decision`, enforced by an import-purity test.
- `policy.Decision` shape: `{Allow bool, Level string ("allow"|"warn"|"block"), Reason string, RuleIDs []string}`.

## Tasks

### Task 1 — pkgparse: capture the remote-source spec
In `internal/pkgparse/pkgparse.go`:
- Add a field to `ParsedCommand`: `RemoteSource string` — the kind of non-registry source, one of `""` (normal registry install), `"git"`, `"url"`, `"tarball"`, `"github"`, `"file"`. Keep `Raw`/`Manager`/`Ecosystem`/`Verb`/`Package`/`Version`/`IsInstall`/`IsExec`/`Sudo`/`Unpinned` unchanged.
- Add a pure helper `classifyRemoteSource(token string) string` that, given the raw install token (before `splitVersion`), returns the kind. Detection (case-insensitive, prefix/suffix string checks only — NO net/regexp needed, keep it pure):
  - `"git"`: token starts with `git+`, `git://`, or ends with `.git`, or starts with `git@` (scp-style).
  - `"github"`: token starts with `github:`, `gitlab:`, `bitbucket:` (host shorthands). (npm/pnpm/yarn accept `github:owner/repo`.)
  - `"url"`: token starts with `http://` or `https://` (and is not already classified git/tarball).
  - `"tarball"`: a `url` whose path ends with `.tgz`, `.tar.gz`, or `.tar` → classify as `"tarball"` (a URL subtype; pick tarball over url when both match).
  - `"file"`: token starts with `file:` or (npm) `./`, `../`, or an absolute path that is clearly a local spec (`/` or a Windows `X:\`). Be conservative: only `file:` and `./`/`../` prefixes to avoid false positives on package names.
  - else `""`.
- Wire it into `parseSegment`: compute `RemoteSource` from the original `token` (the first package token / `-p` flag value) BEFORE `splitVersion`, so a `git+https://…@v1` ref does not get mangled into Package/Version. When `RemoteSource != ""`, leave `Package` as the normalized token if it still resolves a name, but do not require it.
- Pure: pkgparse imports only `strings`. Do NOT add net/url/regexp (would break `TestPkgparseImportsArePure`).

### Task 2 — policy: EvaluateRemoteSource (pure)
New file `internal/policy/remote_source.go`, mirroring `release_age.go`:
- `const ruleRemoteSource = "remote-source-policy"`.
- `type RemoteSourceInput struct { Ecosystem, Package, Kind string }` where `Kind` is the `ParsedCommand.RemoteSource` value.
- `type RemoteSourceConfig struct { Exclude []string }` (allowlist of package/spec names exempt) + `DefaultRemoteSourceConfig()`.
- `func EvaluateRemoteSource(input RemoteSourceInput, cfg RemoteSourceConfig) Decision`:
  1. If `input.Kind == ""` → allow (normal registry install, "no remote source").
  2. If `input.Package` in `cfg.Exclude` → allow ("remote-source allowlisted").
  3. Else → WARN, Reason names the kind, e.g. `"install pulls from a %s source (%s); flagged for review"`. Level `"warn"`, `Allow:true` (warn does not block), `RuleIDs:[ruleRemoteSource]`.
- The default is warn, NOT block (PRD Layer 1). The block-raising is roadmap (do not add per-rule severity config here).
- Pure: imports only `fmt`/`strings`.

### Task 3 — tests
- `internal/pkgparse/pkgparse_test.go`: table cases for each kind — `npm install git+https://github.com/a/b.git`, `npm i github:a/b`, `pnpm add https://example.com/p.tgz` (→ tarball), `npm i https://example.com/p` (→ url), `npm i file:../local` (→ file), `npm i ./local` (→ file), and a normal `npm i left-pad@1.0.0` (→ ""), plus a scoped `@scope/pkg@1.0.0` stays "" (no false positive on the leading `@`). Assert `RemoteSource` and that a normal scoped install still parses Package/Version correctly (regression guard).
- `internal/policy/remote_source_test.go`: each kind warns; `""` allows; allowlist exempts; assert Level/Allow/Reason/RuleIDs. Add an import-purity assertion mirroring the other `_test.go` purity tests in the package.
