<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Playwright webServer → static out/ | A local `serve` process exposes out/ on 127.0.0.1:4199 for the test run only (loopback, ephemeral) |
| chromium (test browser) → served pages | Headless browser drives the real static build; no external network beyond loopback |
| built out/ files → Node test process | gfx.spec.ts + seo.test.ts read out/*.html from disk (repo-local, no user input) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-05 | Information Disclosure | webServer binding | mitigate | Bind loopback only via `serve out --listen 4199` (127.0.0.1); reuseExistingServer:false in CI; ephemeral — torn down at end of run |
| T-19-06 | Tampering | `pnpm dlx serve` fetching the serve CLI | accept | `serve` (by Vercel) is the established `pnpm start` server already vetted for this project (Phase 11+); dlx resolves a pinned-by-lockfile transitive at install; no new tracked dep |
| T-19-07 | Denial of Service | webServer port collision (Pitfall 3) | mitigate | Port pinned to 4199 with `--listen`; webServer.timeout 15000 fails fast rather than hanging |
| T-19-08 | Information Disclosure | gfx/seo file reads of out/ | accept | Read-only over repo-local build artifacts; no secrets in static HTML; pure dev-tooling |
</threat_model>
