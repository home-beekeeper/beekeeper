# BLOCKING CONSTRAINTS — Read Before Anything Else

> Discovered through actual failure this session. Acknowledge before proceeding.

- [ ] CONSTRAINT: **Don't make the live hook fail on every call = lockout** — beekeeper runs `beekeeper check` (fail-closed) on EVERY tool call. A failed COMPILE is safe (`go install` won't replace the binary on a build error). The lockout risk is a binary/config that makes the check fail on ~every invocation: (a) a panic/hang in an always-run path (config load, startup, core eval) → fail-closed → ALL tools (incl. Edit/Read) denied; (b) a config the LIVE binary's validator REJECTS — e.g. setting `nudge.mode=block` while an old binary that only knows soft/hard is still the hook → config-load fails → fail-closed every call. **Order matters: install the new binary BEFORE the config that depends on it.** Smoke-test (`go build -o /tmp/bk.exe ./cmd/beekeeper` + pipe a check) only if you touched startup/config/eval code. A bug on SOME inputs just blocks those calls (annoying, not a lockout).
- [ ] CONSTRAINT: **Block mode is LIVE — your own unquoted compound installs get DENIED** — `nudge.mode=block` is active in `%APPDATA%/beekeeper/config.json`. An UNQUOTED `npm install` / `cd x && npm install` in a Bash command you run is blocked by the hook. Quoted install text in args (e.g. a commit message) is fine (quote-aware). If a `git commit -m` ever trips it, use `git commit -F <file>` (the hook inspects only the command string, not file contents).

**Do not proceed until both boxes are checked.**

## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Code-default-block cascade | Flipping `DefaultNudgeConfig().Mode` to "block" cascaded ~12 test failures across check+gateway, because `npm install` is the universal test command (block-everywhere). | blocking | If you DO want the hard code default (task #10), expect to make ~12 soft-explicit test updates AND accept npm blocking in gateway/library/programmatic contexts. The chosen alternative is installer-writes-block (commit 87ffd66). |
| Live-hook fails every call | A binary/config that makes `beekeeper check` fail on ~every invocation — a panic/hang in an always-run path, OR a config the live binary's validator rejects — fail-closes ALL tools. (Failed compiles are safe; a bug on only some inputs is not a lockout.) | blocking | Install the new binary BEFORE the config that depends on it; smoke-test (build-to-temp + pipe a check) if you touched startup/config/eval code. |
| Over-split parser false-positive | A separator-aware-but-quote-blind parser blocks commands that merely QUOTE an install (commit messages, echo). | advisory | FIXED (quote-aware splitSegments, 54c6c8a). Residual: intra-word quote tricks (`n''pm install`) still evade — exotic, out of scope. |
