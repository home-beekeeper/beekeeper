<objective>
Build the macOS ingestion layer: the eslogger NDJSON parser, the eslogger subprocess builder + stdout drain goroutine, the launchd plist writer and launchctl wrappers, and the synthetic NDJSON fixtures used by unit tests. All files are `//go:build darwin`. This plan covers SMAC-02 (event ingestion + normalization to the existing `internal/sentry.SentryEvent` struct) and SMAC-04 (coverage gap text used by `beekeeper protect status`). The launchd lifecycle install/uninstall/status are wired into the CLI in 07-03; this plan only provides the helpers.

Purpose: Parsing eslogger NDJSON is the load-bearing risk surface of macOS Sentry — Apple's schema is partially undocumented. Isolating the parser behind a pure function (one NDJSON line → one SentryEvent) makes it unit-testable on Windows from synthetic fixtures, and lets the macos-latest CI job (added in 07-05) validate field names against live eslogger output by feeding real captures into the same parser.

Output: `internal/sentry/darwin/` package containing parser, eslogger supervisor, launchd helpers, and a `testdata/` corpus of four synthetic NDJSON fixtures.

    `func parseEsloggerLine(data []byte) (sentry.SentryEvent, error)`:
    1. Outer Unmarshal into esloggerLine. On json error return wrapped ParseError (fmt.Errorf("%w: %v", ParseError, err)).
    2. Determine WallTime: if line.Time != "" attempt time.Parse(time.RFC3339Nano, line.Time); on parse failure or empty, set wallTime = time.Now().UTC() (do NOT error — partial event is acceptable).
    3. Switch on line.EventType:
       - "exec": Unmarshal line.Event["exec"] into esloggerExecEvent. PID = target.AuditToken.PID; PPID = target.PPID; UID = target.AuditToken.UID; Exe = target.Executable.Path; Cmdline = strings.Join(target.Args, " "). Kind = sentry.EventProcessCreate.
       - "open": Unmarshal line.Event["open"] into esloggerOpenEvent. PID = line.Process.AuditToken.PID; PPID = line.Process.PPID; UID = line.Process.AuditToken.UID; Exe = line.Process.Executable.Path; FilePath = openEv.File.Path. Kind = sentry.EventFileAccess.
       - "create": Unmarshal line.Event["create"] into esloggerCreateEvent. Same process fields as "open"; FilePath = createEv.Destination.ExistingFile.Path. Kind = sentry.EventFileAccess.
       - "network_flow": Unmarshal line.Event["network_flow"] into esloggerNetworkFlowEvent. Same process fields. DstAddr = net.ParseIP(netEv.Remote.Address) (nil-tolerant — pass through whatever ParseIP returns); DstPort = netEv.Remote.Port. Kind = sentry.EventNetworkConnect.
       - default: return zero SentryEvent and fmt.Errorf("%w: unknown event_type %q", ParseError, line.EventType).
    4. Validation: if PID == 0 AND eventType is not "exec" (exec target has its own PID), still allow it (PID 0 is kernel; do not over-reject). However, if PID == 0 AND PPID == 0 AND Exe == "" simultaneously (clearly malformed), return ParseError.

    `func EsloggerCommand(ctx context.Context, events []string) *exec.Cmd`:
    - args := append([]string{}, events...)

    `func drainEslogger(r io.Reader, out chan<- sentry.SentryEvent) error`:
    - scanner := bufio.NewScanner(r); scanner.Buffer(make([]byte, 64*1024), 64*1024).

    - Non-blocking send: `select { case out <- ev: default: atomic.AddUint64(&EventsDropped, 1) }`.
    - After loop: return scanner.Err().
