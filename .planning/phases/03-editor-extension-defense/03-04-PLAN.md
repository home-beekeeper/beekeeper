<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| bumblebee subprocess stdout → scan | Untrusted NDJSON from an external binary crosses into the merged output + audit log |
| user terminal → init/purge prompts | Consent input crosses into destructive/settings-mutating actions |
| quarantine CLI arg → quarantine package | Attacker-influenced id crosses into a filesystem operation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04-01 | Tampering | Compromised/garbage bumblebee stdout crashing or poisoning the scan | mitigate | Each NDJSON line is parse-validated; invalid lines are skipped with a structured warning, never crash the scan; unknown record_types pass through unmodified but the process never trusts stdout to control flow. |
| T-03-04-02 | Denial of Service | bumblebee subprocess hanging | mitigate | exec.CommandContext bound to the command context; the buffered channel has a fixed cap; ctx cancellation (Ctrl+C) terminates the subprocess. |
| T-03-04-03 | Tampering | quarantine restore/purge id traversal via CLI arg | mitigate | The CLI passes the raw arg straight to quarantine.Restore which applies filepath.Base + prefix guard (T-03-02-01); no path construction in main.go. |
| T-03-04-04 | Elevation | init writing editor settings without consent | mitigate | Per-editor, per-action explicit consent prompts; --yes is an explicit opt-in for non-interactive use; --no-editors preserves the zero-touch Phase 1 behavior. Settings write is atomic + idempotent. |
| T-03-04-05 | Repudiation | quarantine restore/purge with no audit trail | mitigate | restore and purge each emit an NDJSON audit record (rule_ids ["EDXT-05"]) before/after the action. |
| T-03-04-06 | Tampering | Forbidden `--format ndjson` flag breaking the bumblebee subprocess | mitigate | Hard gate in Task 1 acceptance criteria: the string "scan --format ndjson" must be absent; `bumblebee scan [--profile deep]` only. |
</threat_model>
