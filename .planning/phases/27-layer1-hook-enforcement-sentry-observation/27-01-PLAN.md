# Plan 27-01 — Install posture at the hook

## Goal

Wire the three install-posture rules into the pre-exec hook (`beekeeper check` →
`internal/check/handler.go`), where the nudge block used to be, so that an
agent-issued install is evaluated against the default posture and WARNED (not
blocked) when it is suspiciously fresh, carries lifecycle scripts, or pulls from
a git/remote/URL/file source. This is PRD Layer 1.

## Design decisions (Gate-1 load-bearing — present these exactly at Gate 1)

1. **Default action = WARN for all three rules.** The PRD default posture warns; it
   does not block. The pure evaluators `EvaluateReleaseAge` and `EvaluateLifecycle`
   return `block` on a violation (correct for the scan/watch EXTENSION path, which
   is unchanged). The hook posture adapter maps any rule "fired" outcome to a WARN
   decision (`Allow:true, Level:"warn"`). Raising a rule to block per ecosystem is
   roadmap (deep per-rule editing); do NOT add a severity knob now.
2. **Fail-soft on unknown/timeout, not fail-closed.** Resolving release-age and
   lifecycle requires a registry fetch. On a missing publish timestamp, an
   unsupported ecosystem, a registry error, or a fetch timeout, the posture rule
   WARNS ("release age unknown" / "lifecycle scripts unknown"), it does NOT block.
   Rationale: the PRD's explicit goal is "do not break the build"; the primary
   malware defense is the corroboration catalog block (separate, still fail-closed
   and still merged most-restrictive). Posture is an advisory early-warning, so a
   registry outage must not turn into a blocked install. THIS IS A DELIBERATE
   DIVERGENCE from the pure evaluators' fail-closed block (which remains for the
   extension scan/watch path). Flag it explicitly at Gate 1.
3. **Latency bound.** Posture runs ONLY on parsed install commands (rare vs file
   reads), never on every tool call. Wrap the registry fetches in a short child
   context (derive from the handler ctx, cap ~2.5s) so posture can never blow the
   hook budget; on deadline, warn-unknown (per decision 2) and move on. Release-age
   is cache-first (`FetchPublishAge`, 24h TTL); add an equivalent disk cache for
   lifecycle (mirror `age_cache.go`) so repeat installs are fast and the registry
   is not hammered.
4. **Most-restrictive merge preserved.** Posture contributes warn-level decisions
   merged via `mergeDecisions` after self-protection, exactly where the nudge block
   was. A catalog `block` or sensitive-path `block` still wins over a posture warn.
   Posture can never DOWNGRADE a block.

## Tasks

### Task 1 — export a cached lifecycle adapter in internal/catalog
- Export `FetchLifecycleScripts(ctx, client, ecosystem, pkg, version) ([]string, error)`
  (wrap the existing unexported `fetchLifecycleScripts`).
- Add a disk cache mirroring `age_cache.go` (`lifecycle_cache.go`): cache-first with
  a 24h TTL keyed by ecosystem/pkg/version; cache both the script list and the
  "unsupported/failed" outcome (so non-npm ecosystems and registry errors are not
  re-fetched within the TTL). Return shape lets the caller build `LifecycleInput`
  with `ScriptsPresent` or `RegistryCheckFailed`.
- Unit tests mirroring `age_cache_test.go` (fresh, cache-hit, error→cached-failure,
  stale-refetch).

### Task 2 — the posture adapter (internal/check/posture_adapter.go)
Mirror the structure of the deleted `nudge_adapter.go`. Add:
`evaluatePosture(ctx, tc policy.ToolCall, cfg config.Config, client *http.Client, cacheDir string, now time.Time) (policy.Decision, []policy.Decision, bool)`
(or a single merged decision + audit detail — keep it simple).
- Skip (ok=false) when: `tc.ToolName != "Bash"`; no/empty command; `pkgparse.Parse`
  reports `!IsInstall`.
- For the parsed install:
  - **remote-source** (always, pure/instant): `policy.EvaluateRemoteSource({Ecosystem, Package:parsed.Package or parsed.Raw spec, Kind:parsed.RemoteSource}, DefaultRemoteSourceConfig())`. Already returns warn.
  - **release-age** (only when `parsed.RemoteSource == "" && parsed.Package != ""`): a child ctx with the ~2.5s cap; `catalog.FetchPublishAge(...)` → build `ReleaseAgeInput` → `policy.EvaluateReleaseAge(...)`. Map result via `posturize()` (Task 3). On `missing`/err/timeout → warn-unknown.
  - **lifecycle** (only for registry installs): `catalog.FetchLifecycleScripts(...)` (cached) → build `LifecycleInput` (set `RegistryCheckFailed` on unsupported/err) → `policy.EvaluateLifecycle(...)` → `posturize()`. On unsupported/err/timeout → warn-unknown.
- Combine the per-rule decisions with `mergeDecisions` (all warn-capped, so the
  result is warn if any rule fired, else allow). Reason should name what fired
  (e.g. "install posture: package age 30m below 1440m; lifecycle scripts present").
- Pure-w.r.t.-caching is not required (this is the impure adapter, like the old
  nudge adapter and `paths.go`).

### Task 3 — the warn mapping
- `posturize(d policy.Decision) policy.Decision`: if `d.Allow` (rule did not fire)
  return d unchanged (allow). Otherwise return `{Allow:true, Level:"warn", Reason:d.Reason, RuleIDs:d.RuleIDs}` — the default posture action. Add a code
  comment citing PRD Layer 1 (warn default) and the enforcement boundary.

### Task 4 — wire into the handler
- In `internal/check/handler.go`, replace the Phase-26 NOTE comment block with the
  posture call: `decision = mergeDecisions(decision, evaluatePosture(ctx, toolCall, cfg, client, cacheDir, now))` at the same position (after self-protection, before the final deadline check). Use the handler's existing http client / cacheDir / now (or construct a 4s-timeout client like scan does; the per-fetch child ctx is the real bound).
- Add a top-of-block comment carrying the enforcement-boundary statement (IPBND-01):
  posture is enforced pre-exec at the hook for hooked Tier-1 harnesses only and
  inherits their tier caveats; installs run outside a hooked tool call are observed
  and audited by Sentry, not prevented here; the shim and gateway are not general
  install surfaces; machine-wide human-install pre-exec enforcement is roadmap.

### Task 5 — tests (BOTH unit AND live-path)
- `posture_adapter_test.go`: inject the fetchers via seams (package-level fn vars,
  mirroring the old `nudge.DetectStateFn` seam) so tests run without a real
  registry. Cover: fresh package → warn(age); old package → allow; lifecycle
  scripts present → warn; git/url/file install → warn; missing-timestamp →
  warn-unknown (NOT block); registry error → warn-unknown; clean registry install
  → allow; non-Bash / non-install → skip.
- **RunCheck-level integration test** (the live hook path, per the "test the PATH
  not the component" rule): drive `RunCheck`/`RunCheckTo` with a Bash `npm install`
  tool call against a stubbed registry/cache so a <24h package produces a warn in
  the returned decision AND the audit record, exit code 0 (warn does not block).
  Also assert a catalog-blocked package still BLOCKS (posture warn cannot downgrade
  a block).
