<objective>
Build the catalog watch daemon (CTLG-06) and catalog sanity bounds with degraded mode (CTLG-08). The watch loop polls catalog sources on a configurable interval, detects deltas against persisted state, runs the sanity check, and fires a delta callback (the audit-recording + re-scan trigger is wired by the caller in Plan 08).

Purpose: A new threat-intel entry should be acted on within the poll interval without the developer running anything. Sanity bounds are a self-defense non-negotiable (CLAUDE.md Phase 2): a catalog source that suddenly balloons by thousands of entries is treated as suspect and degraded to warning-only rather than trusted to block.

Output: `internal/catalog/sanity.go`, `watch.go`, `state.go` (+ tests). Depends on Plan 04 + Plan 05 because the watch loop polls all enabled sources and the OSV/Socket adapters must exist to be polled. The CLI wiring (`catalogs watch`, `catalogs verify`) is in Plan 08 to avoid a main.go file conflict.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| catalog source → watch loop | A poisoned upstream catalog could inject thousands of fake block entries in one sync |
| watch state file → loop | state.json under ~/.beekeeper could be tampered to hide a prior degraded mark |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-07-01 | Tampering | catalog feed poisoning via mass entry injection | mitigate | CheckSanity hard-blocks deltas >10000 and alerts >1000, degrading the source to warning-only — a poisoned source cannot suddenly block thousands of packages (the "2FA principle" for catalog feeds, CLAUDE.md §12.3) |
| T-02-07-02 | Denial of Service | watch loop tight-spins on persistent sync error | mitigate | Errors log-and-continue on the ticker cadence, not a busy loop; PollInterval clamped ≥5m |
| T-02-07-03 | Denial of Service | goroutine leak on shutdown | mitigate | Watch returns on ctx.Done(); TestWatchExitsOnCancel proves prompt clean exit; ticker stopped via defer |
| T-02-07-04 | Tampering | state.json hiding a prior degradation | accept | state.json is owner-only under ~/.beekeeper; local tamper implies broader compromise; `catalogs verify` (Plan 08) lets the user re-baseline deliberately |
</threat_model>
