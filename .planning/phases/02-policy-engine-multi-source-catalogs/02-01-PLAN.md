    Create `corroboration.go` in package policy. Implement `func corroborate(matches []CatalogMatch, t CorroborationThresholds) (level string, quarantine bool, count int, agreed, dissented []string)`.

    Logic: build a set of distinct SIGNED `CatalogSource` values and a set of distinct UNSIGNED source values. `signedCount := len(signedSet)`. Enforcement requires `hasSignedSource := signedCount >= 1`. Escalation by signed source count only — unsigned sources contribute 0.5 weight and can never alone reach BlockAt (model this as: unsigned sources only ever justify "warn"). Decision table:
      - `signedCount >= t.QuarantineAt && hasSignedSource` → ("block", true)
      - `signedCount >= t.BlockAt && hasSignedSource` → ("block", false)
      - `signedCount >= t.WarnAt || hasUnsigned` → ("warn", false)
      - else → ("allow", false)
    `count` is `signedCount`. `agreed` is the sorted distinct list of all matched source names (signed and unsigned). `dissented` is empty for now (no dissent model in Phase 2 — a source that returns no match simply does not appear; reserve the field). Use only the `sort` and `strings` stdlib packages — both are pure and NOT in the forbidden set. Do not import `time`/`os`/`net`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| agent → policy engine | ToolInput is attacker-influenceable; package coordinates extracted defensively |
| catalog source → corroboration | A poisoned or spoofed catalog source could fabricate matches to force false blocks (DoS) or, inversely, withhold matches |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01-01 | Tampering | corroborate() unsigned-source weighting | mitigate | Unsigned sources never reach block alone — `hasSignedSource` gate requires ≥1 signed source for any block/quarantine; proven by TestEvaluateUnsignedNeverBlocks |
| T-02-01-02 | Denial of Service | corroborate() false-positive flooding | mitigate | "Independent" = distinct CatalogSource; same source duplicated counts once (TestCorroboration same-source-twice). Caps the blast radius of one poisoned source to "warn" |
| T-02-01-03 | Spoofing | Decision provenance fields | mitigate | SourcesAgreed records exact catalog_source names so a downstream audit (Plan 08) can attribute every block to named sources; no anonymous escalation |
| T-02-01-04 | Tampering | hidden I/O / policy drift | mitigate | TestCorroborationImportsArePure + TestEngineImportsArePure enforce the pure-library import allowlist at AST level |
</threat_model>
