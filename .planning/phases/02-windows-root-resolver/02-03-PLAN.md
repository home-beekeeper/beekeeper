| Built pollen binary subprocess → test | runBinaryOnFixture execs the freshly built binary against the fixture only |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-02-07 | Tampering | Fixture reaching into real user data | mitigate | Fixture is a committed `testdata/parity-fixture/` tree scanned via explicit `--root`; no default Windows/Unix root discovery runs, so no real %USERPROFILE%/$HOME path is ever walked. | T-02-09 | Information Disclosure | Cross-OS parity false-pass hiding detector drift | accept | Per-OS coverage assertion + identical test on all 3 runners is the parity signal; exact byte-equality across OSes is intentionally not required (OS path strings differ by design — PTEST-01 wording). Low risk: a real detector regression drops an ecosystem and fails coverage on every OS. |
