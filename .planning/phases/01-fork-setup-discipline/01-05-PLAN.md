|----------|-------------|

| published release → downstream operator/beekeeper | Consumers trust the cosign signature + SBOM to confirm artifact integrity and provenance. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-01-18 | Repudiation | Releasing without provenance/SBOM | mitigate | Task 2 asserts the release carries checksums.txt.sigstore.json, *.cdx.json, and *.intoto.jsonl; Task 3 confirms the SBOM records the pinned upstream SHA (SDEF-02). |
| T-01-19 | Elevation of Privilege | A single compromised account cutting an unauthorized signed release | mitigate | Two-account release-approval discipline at the tag step (release_facts); the human checkpoint confirms it was followed. |
| T-01-20 | Tampering | Tagging on a red/unknown CI matrix (shipping unverified guards) | mitigate | Task 1 is a BLOCKING human-verify checkpoint: tagging cannot start until a human replies CI-GREEN confirming the full 3-OS matrix + differential + govulncheck are green.
