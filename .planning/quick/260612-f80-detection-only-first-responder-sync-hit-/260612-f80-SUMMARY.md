### Task 4 (A+C1) â€” First-responder + TUI incident (`ce880ed`)

Two new files:

**`internal/watch/firstresponder.go`** â€” `RunFirstResponder(ctx, FirstResponderConfig) error`:
- Calls CrossReference to get catalog hits
- For each hit: always calls `targets.AddTarget()` (sentry target-list record)
- Enabled + threshold met + PathResolved: calls `quarantine.MoveTyped()` + writes `catalog_quarantine` audit
- Enabled + threshold met + !PathResolved: writes `pending-quarantine` audit (fail-safe)
- DryRun: writes `would-quarantine` audit only
- Below threshold: no audit record, no action
- Move error: logs `quarantine_error` + continues (artifact stays in place, fail-closed)
- After loop: `sentry.SaveTargets()` best-effort

**`internal/tui/incidents.go`** â€” `CatalogQuarantineIncidentFromRecord()`:
- quarantined hit: [R]estore, [P]urge (irreversible, danger class), [A]cknowledge
- pending hit (path unknown): [A]cknowledge only
- Purge is NEVER auto-triggered (user-gated)

## Honesty Invariants Verified

- **Scan READ-ONLY:** `CrossReference` calls pollen, reads records, evaluates via policy â€” never removes, disables, or edits installed packages
- **Quarantine REVERSIBLE:** `MoveTyped` uses `os.Rename` + manifest JSON; `Restore()` uses `os.Rename` back. The destructive `Purge` is NEVER called automatically (user-gated via CLI/TUI)
- **Sentry DETECTION-ONLY:** `TargetList` only lowers correlation thresholds (CredAccessThreshold/CredCLIThreshold â†’ 1); no kill/isolate/network-cut field added to `SentryAlert`; `TestSentryAlertHasNoDestructiveAction` verifies the struct
- **Fail-closed:** `MoveTyped` error â†’ `quarantine_error` audit + artifact left in place; pollen unavailable â†’ empty hits (no panic); `LoadTargets` missing file â†’ empty list (not error)
- **`internal/policy` stays pure:** All threshold comparisons live in `internal/watch` (first-responder) and `internal/sentry` (rule tightening); no I/O, goroutines, or side effects added to policy package
