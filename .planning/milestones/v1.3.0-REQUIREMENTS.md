**Defined:** 2026-06-07

**Core Value:** A hijacked or off-task agent cannot successfully act on the developer's machine without Beekeeper deciding to permit it.
**Milestone goal:** Ship Beekeeper's public-facing Next.js site — a distinctive marketing home and a complete documentation set — that takes a developer from "what is this" to installed, configured, and confident in its security posture. (Expanded mid-cycle to include the Go-core pre-ship validation gate — Phases 20–21.)

> Greenfield web app under `web/` (in-repo, pnpm workspace, isolated from the Go module). Static export (Next.js 16 / Tailwind v4 / shadcn/ui / Fumadocs / React-Three-Fiber).

## Future Requirements (deferred)

- Interactive in-browser playground / tool-call decision demo (sandboxing an OS-hook security tool is infeasible and trust-risky)
- Per-page dynamic OG images (`next/og`) — single static OG image in v1.3.0
- Versioned docs (multiple product versions); i18n / multi-language docs; community / blog section
- Auto-generated CLI reference from the Go binary (hand-authored MDX in v1.3.0)

## Out of Scope (explicit exclusions)

- **Interactive demo / playground** — cannot safely sandbox a filesystem + OS-hook tool in the browser
- **Blog**; **AI chatbot over docs** (hallucination risk for security content); **SSR / ISR / server runtime** (static export only, locked); **CMS / headless backend**
- **Changing the Go product** — web phases (11–19) are web-only. **Exception: Phase 21 (Full-System Validation)** adds Go test coverage + a cross-platform CI matrix and MAY apply coverage-surfaced fixes, as the pre-ship release gate (added 2026-06-10; zero product behavior changed in practice)
