<objective>
Land the supporting plumbing that downstream consumers (Plans 04/06/07/08) depend on, none of which
depend on the nudge decision types — so this runs in Wave 1 in parallel with Plan 01:

1. **`platform.StateDir()` BEEKEEPER_HOME override** — the confirmed Wave 0 blocker (research A2 /
   Open Q1). `StateDir` currently resolves via `os.UserConfigDir()`/`os.UserHomeDir()` with NO
   env override, so the compiled binary cannot redirect state/audit to a temp dir. The BTEST-03
   live-binary E2E (Plan 06/07) REQUIRES this to be hermetic. Add the override BEFORE the E2E task.
2. **`audit.AuditRecord` nudge fields + record types + the §9 `nudge_action` field** (NUDGE-06,
   BLOCKER 2) — additive, omitempty-tagged. The repo's existing `Decision` field is `allow|warn|block`;
   PRD §9 demands a `decision` enum of `advise|proceed|rewrite|block`. To preserve §9 semantics
   WITHOUT disturbing the existing field, add an explicit `NudgeAction` field (closed enum
   advise|proceed|rewrite|block) populated from `nudge.ActionString`. A §10-14 conformance test
   asserts the COMPLETE §9 field set + a closed-enum reason_code + the correct nudge_action.
3. **`config.NudgeConfig` + layered defaulting + an EXPORTED `ValidateNudgeConfig` fail-closed
   bounds validator** (NUDGE-08, WARNING 3, plus the §10-17 cross-package need) — `config` defines
   its OWN NudgeConfig struct (config imports only stdlib — no import of internal/nudge, avoiding a
   cycle); a missing block resolves to documented defaults; project `.beekeeper.json`
   `nudge.enabled:false` wins; AND an invalid block (bad mode, unknown preferred, malformed floor) is
   rejected at load with a non-nil error (fail-closed, mirroring the `fail_mode` switch and
   `validateCorroborationThresholds`). The validator is EXPORTED as `func ValidateNudgeConfig(NudgeConfig) error`
   because `cmd/beekeeper` (package `main`, Plan 08) must call it for the §10-17 `config set` rejection
   test and cannot reference an unexported symbol in `internal/config`; the internal `Load` path
   delegates to the same exported function so there is ONE validator.

Purpose: Decouple the E2E-enabling env override, the audit schema extension (with the §9 nudge_action
fix), the config block, and its fail-closed bounds validation from the nudge decision core so they
can be built first and in parallel — this removes the A2 blocker the live-binary release gate hangs on,
and exports the single validator Plan 08's `config set` consumes.
Output: `internal/platform/dirs.go`, `internal/audit/types.go`, `internal/config/config.go` + tests.
</objective>
