<objective>
Land the v1.2.0 behavioral test battery — the milestone's primary ask — proving the runtime wiring
is live, not just that component functions return correct values:

1. **BTEST-02 integration tests** — extend `internal/check/integration_test.go` (the existing
   `runCheckWithIndex` + `readLastAuditRecord` harness) with cases that drive raw stdin JSON for a
   pnpm/bun install and assert BOTH the exit code AND the audit NDJSON record. Include a pnpm
   install of a catalog-malicious package (keyed `npm::evil-pkg`) proving F3 is closed end-to-end,
   and a soft-mode pnpm-available advisory (exit 0, record_type "nudge"). The advisory case injects
   detection by swapping the EXPORTED `nudge.DetectStateFn` seam (Plan 04) — these tests live in
   package `check`, which CANNOT assign the unexported `nudge.pnpmVersionFn`/etc., so the exported
   `DetectStateFn` seam is the only cross-package injection point.
2. **BTEST-03 live-binary E2E** — NEW `internal/check/e2e_test.go` (`//go:build e2e`, net-new, no
   repo precedent). Build the compiled binary, run `beekeeper check` against the real catalog with
   raw stdin for a credential read (SPATH → exit 1), an `ai-figure` critical install (CORR →
   exit 1), and a `pnpm add` / `bun add` install (NUDGE → parsed + audited). Assert exit codes AND
   read the audit NDJSON record_type + decision. This is the release gate — it must pass before any
   v1.2.0 tag is cut.

The E2E is hermetic via the Plan 05 `BEEKEEPER_HOME` override (the resolved A2 blocker). The bun
E2E case must `t.Skip` when `exec.LookPath("bun")` fails (bun is not on the dev box / many CI runners).

Purpose: This closes the loop on the milestone — the same validation run that surfaced F1/F2/F3 is
now an automated, hermetic, repeatable test. The integration tier proves wiring; the E2E tier proves
the shipped binary behaves correctly against the real catalog.
Output: `internal/check/integration_test.go` (extended) + `internal/check/e2e_test.go` (new).
