<objective>
Implement the pure-logic core of corroboration severity escalation (CORR-01) together with its anti-poisoning sanity bounds (CORR-02) as a single atomic deliverable. A `critical`-severity catalog match escalates to **block** at a single trusted source via `SeverityOverrides["critical"]={BlockAt:1}`, gated by `CatalogHealthy`, guarded against all-versions wildcard entries, and bounds-checked so an override can never be `BlockAt < 1` or looser than the global threshold.

Purpose: Close finding F1 — `ai-figure` (Shai-Hulud, OSV `MAL-2026-4126`) is currently warn-only because `corroborate()` counts only signed sources (bumblebee unsigned + OSV signed = signedCount 1 = warn). The escalation MUST never ship without its sanity gate (STATE.md Blockers/Concerns: "shipping escalation without the sanity gate inverts the trust model").
Output: Extended `internal/policy/types.go` + `corroboration.go`, six new RED→GREEN unit tests, and an audited selftest fixture set that stays green under the new defaults.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| catalog source → corroborate() | Catalog match severity/signed fields are attacker-influenceable (a flooded or poisoned catalog can mis-tag entries); they cross into the pure decision engine here |
| policy file → CorroborationThresholds | Operator/agent-writable policy JSON can carry override values that, if unbounded, weaken or invert enforcement |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Tampering | corroborate() severity escalation | mitigate | findSeverityOverride returns nil when CatalogHealthy=false — a flooded catalog (>1000 alert-delta entries → SourceState.Degraded) cannot trigger a single-source block storm. Proven by TestCorroborationDegradedCatalogNoEscalation. |
| T-06-02 | Tampering | mis-tagged all-versions entry | mitigate | findSeverityOverride returns nil when ANY non-dissented match has Version=="*" — a catalog that tags `react`/`typescript` as critical+wildcard still needs 2-source corroboration. Proven by TestCorroborationAllVersionsCriticalWildcardStaysWarn. |
| T-06-03 | Tampering / Elevation | override BlockAt < 1 (blocks unconditionally or zero-threshold) | mitigate | validateCorroborationThresholds rejects BlockAt<1; corroborate fails closed to "block" on the error (fail-closed per CLAUDE.md). Proven by TestValidateCorroborationThresholdsRejectsBlockAtZero. |
| T-06-04 | Spoofing | override looser than global (BlockAt > global BlockAt) | mitigate | validateCorroborationThresholds rejects override BlockAt>t.BlockAt — a more-dangerous severity class can never be made laxer than the baseline. Proven by TestValidateCorroborationThresholdsRejectsLooserOverride. |
| T-06-05 | Information Disclosure | dissent sentinels mis-counted as real matches | accept | findSeverityOverride skips m.Dissented (Phase-11 dissent-sentinel decision); no new disclosure surface in a pure function. |
</threat_model>
