<objective>
Make the v1.2.0 CORR live-binary release gate hermetic. Today `TestE2ELiveBinary/CORR_aifigure_critical_block` seeds an UNSIGNED, `Versions:["*"]` wildcard `ai-figure` entry; corroboration yields only `warn` from that fixture (all-versions wildcard guard + unsigned source suppress the critical escalation), so the case blocks ONLY because the compiled binary reaches live osv.dev and gets a signed match.
