<design_source_notes>
Per CONTEXT D-01, `beekeeper-docs.html` (repo root) is the authoritative design. This plan
codifies its `header.site` (lines 694-713), `.hero` (lines 716-763), and `footer.site`
(lines 1204-1222) blocks into React server components, mapping the mockup's inline CSS onto
the Phase-12 raw theme tokens.

TOKEN RULE (the Phase-12 dual-theme trap): style elements with the raw theme tokens
`var(--bg) / var(--fg) / var(--fg-strong) / var(--dim) / var(--dimmer) / var(--surface) /
var(--surface-2) / var(--border) / var(--border-strong) / var(--amber) / var(--teal) /
var(--coral) / var(--red) / var(--green)`. These are theme-switched in globals.css `:root`
and `.light`. NEVER use the `--color-bk-*` family or `text-bk-*` utilities for color — they
are dark-only constants and freeze the element to dark in light mode (confirmed by
breaking-change-callout.tsx's header comment and 12-03-SUMMARY Deviation 3/4). Use either
`style={{ color: "var(--dim)" }}` or arbitrary Tailwind `text-[var(--dim)]` — both are
dual-theme safe and match the changelog components' established pattern.

DECISION TRACEABILITY: This plan implements D-01 (codify the mockup directly onto Phase-12
tokens) for the chrome + hero. The honest install-command framing implements D-03 (content
accuracy: the install command is canonical but not yet resolvable — repo unpushed).
</design_source_notes>
