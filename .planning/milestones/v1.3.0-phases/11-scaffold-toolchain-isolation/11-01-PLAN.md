<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| npm registry тЖТ developer machine | `create-next-app` / `pnpm install` pull third-party package code that executes lifecycle scripts on the dev machine |
| Node toolchain (web/) тЖТ Go module (repo root) | A web-side install/postinstall must never reach or mutate Go-module files (go.mod, go.sum, cmd/, internal/) тАФ cross-toolchain integrity |
| Working tree тЖТ git history | Build artifacts and the pnpm virtual store must not be committable (integrity/noise + accidental secret-or-artifact leakage) |

> No authentication, session, network-facing server, user input, secrets, or data persistence exist in a static scaffold (per 11-RESEARCH.md "Security Domain": ASVS V2тАУV6 do not apply). The realistic surface is supply-chain / cross-toolchain integrity only тАФ kept proportionate below.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01 | Tampering | npm registry packages installed by create-next-app/pnpm | mitigate | All packages are the canonical Next.js scaffold set (next/react/react-dom/typescript/tailwindcss/@biomejs/biome) from the vercel/meta/microsoft/tailwindlabs/biomejs orgs, version-verified against the npm registry 2026-06-07 (RESEARCH "Package Legitimacy Audit", all [OK]). Commit `web/pnpm-lock.yaml` so the exact resolved tree is pinned and reproducible. No manual `pnpm add` of unaudited packages this phase. |
| T-11-02 | Tampering | Cross-toolchain: a web-side postinstall mutating Go-module files | mitigate | Task 3 asserts `git status --porcelain go.mod go.sum` is EMPTY after `pnpm install` (SC3). pnpm only writes Node directories + `web/pnpm-lock.yaml`; the assertion is a hard, automated gate тАФ a postinstall touching go.mod/go.sum would fail the build before commit. |
| T-11-03 | Tampering / Info disclosure | Build artifacts + pnpm virtual store committed to git | mitigate | Root `.gitignore` adds `node_modules/`, `web/.next/`, `web/out/`, `web/.source/`; create-next-app's `web/.gitignore` covers `/node_modules`, `/.next/`, `/out/`.
