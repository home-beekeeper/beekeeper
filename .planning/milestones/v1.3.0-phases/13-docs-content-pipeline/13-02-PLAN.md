<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| build ‚Üí out/api/search/index.json (public CDN asset) | The static search index ships publicly; it must contain only in-repo doc text, no secrets or absolute source paths |
| in-repo MDX ‚Üí fumadocs-mdx compiler | Content authored in-repo is compiled to RSC at build; no remote/dynamic MDX is fetched |
| RootProvider theme control ‚Üí next-themes | Two theme owners could desync; the boundary must keep next-themes as sole owner |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-03 | Information Disclosure | out/api/search/index.json (public) | mitigate | The index is generated by `createFromSource(source)` exclusively from `loader()` over in-repo `content/docs` text + frontmatter + TOC ‚Äî no env vars, secrets, or filesystem paths are fed in. Acceptance: after build, the emitted `out/api/search/index.json` is reviewed to contain only doc-derived content (titles/headings/body text), no absolute paths or secret-looking tokens. (Seed content in 13-03 is deliberately benign placeholder text.) |
| T-13-04 | Tampering | RootProvider ‚Üî next-themes theme ownership | mitigate | `theme={{ enabled: false }}` on RootProvider disables Fumadocs' internal next-themes instance so the Phase-12 `<ThemeProvider>` remains the single owner ‚Äî prevents a second theme controller from being injected into the provider tree (double-toggle / FOWT). Acceptance: providers.tsx asserts `enabled: false` is present. |
| T-13-05 | Tampering | in-repo MDX ‚Üí static HTML | accept | All MDX is hand-authored in-repo; fumadocs-mdx fetches no remote content and runs no untrusted scripts at compile. No external content-injection vector exists in a server-less static export. Residual risk is low and accepted; content-accuracy review is Phase 18's gate. |

**Disposition note:** No threat in this plan is rated `high`. The one Information-Disclosure item (public search index) is mitigated by construction ‚Äî the index is built solely from in-repo doc text via `createFromSource` with no secret inputs ‚Äî and confirmed by reviewing the emitted JSON. No blocking human checkpoint required.
