<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| seed MDX (in-repo) → out/api/search/index.json (public) | Seed text is indexed and shipped publicly in the search index |
| seed MDX → static HTML (out/docs/) | Hand-authored MDX compiled to public static pages |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-06 | Information Disclosure | seed MDX content in public out/ + search index | mitigate | Seed prose is deliberately thin, generic placeholder text — no secrets, credentials, internal paths, or unreleased details. Acceptance: seed files contain only public command examples (`go install`, `beekeeper check`) and benign placeholder prose; no env vars / tokens / absolute filesystem paths. |
| T-13-07 | Tampering | content-accuracy of security docs | accept | Seed content is clearly labeled placeholder; accurate security-posture claims + `source_doc:` citation + THREAT-MODEL.md review are the Phase-18 (DOCS-09) gate, not this phase. Residual risk (placeholder text mistaken for authoritative) is bounded by the explicit placeholder labeling and accepted for the skeleton phase. |
| T-13-08 | Tampering | MDX → static build | accept | All MDX is hand-authored in-repo; fumadocs-mdx fetches no remote content and executes no untrusted scripts during compile. No injection vector in a server-less static export. Low residual risk, accepted. |

**Disposition note:** No threat in this plan is rated `high`. The Information-Disclosure surface (public seed text + search index) is mitigated by authoring only benign, public placeholder content with no secrets or internal paths. Content-accuracy is explicitly Phase-18's gate and accepted here under placeholder labeling. No blocking human checkpoint required.
</threat_model>
