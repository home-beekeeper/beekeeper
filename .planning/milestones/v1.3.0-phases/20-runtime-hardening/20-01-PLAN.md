<objective>
Tier 1 — Background catalog sync + TUI scheduler (CSYNC). Today a hook-only user
never refreshes threat intel unless they manually run `beekeeper catalogs sync`,
and the TUI `s sync all` key is a dead no-op toast. This plan adds a config-driven
sync interval (project-layer cannot disable it — it is a security-relaxing lever),
ETag-conditional + interval-gated sync, an unprivileged per-OS `catalogs daemon`
that runs `catalogs sync` on an hourly heartbeat, and wires the TUI to a real sync
plus a schedule selector.

Purpose: A hijacked agent's most dangerous window is when the catalog is stale.
Automatic, self-defending background refresh closes that window without elevation.

Output: CatalogSyncConfig + validator + layered merge; SourceState timestamps +
ETag; conditional/interval-gated Sync; `catalogs daemon install|uninstall|status`
(systemd --user / LaunchAgent / schtasks); live TUI sync + selector.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| project/.beekeeper.json + env -> config merge | Untrusted layer (an agent can write a repo file or set env) crosses into the security config. |
| GitHub Contents API -> catalog index | Remote threat-intel content crosses into the local policy engine. |
| OS scheduler -> `beekeeper catalogs sync` | An unprivileged user-level job invokes the binary on a timer. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-20-01 | Tampering/EoP | project/env config layer disabling sync | mitigate | `mergeCatalogSyncUntrusted` refuses `enabled:false` and interval-loosening from untrusted layers (CSYNC-04); trusted user layer may opt out. |
| T-20-02 | DoS | malformed/oversized GitHub list response | mitigate | Sync returns early on parse/HTTP error and preserves last-good index (never pre-deletes); LastError recorded for honest TUI surfacing. |
| T-20-03 | Tampering | swapped catalog content on refresh | accept (covered elsewhere) | Catalog signature/corroboration verification is the existing catalog layer's job; this plan does not weaken it. Conditional 304 path does no rebuild. |
| T-20-04 | EoP | sync daemon requesting elevation | mitigate | Daemon is strictly unprivileged (no `/ru SYSTEM`, no `/rl HIGHEST`, systemd `--user`, LaunchAgent gui domain) per D-T1-host. |
</threat_model>
