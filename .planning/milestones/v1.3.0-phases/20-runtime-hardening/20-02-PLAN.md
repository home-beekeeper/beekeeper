<objective>
Tier 2 — Make the LlamaFirewall opt-in actually work (LLMF). Today prompt scanning
is a SILENT fail-open no-op even on Linux: the sidecar builds `LlamaFirewall()` with
no scanners and calls `UserMessage(role="user", ...)` (a TypeError that is swallowed
into `result="clean"`), and re-constructs the firewall per request. CodeShield is a
stub, AlignmentCheck routes agent context to a 3rd-party cloud (Together AI), the
Windows IPC is a separate named-pipe fork, and `enable/disable/status` read `$HOME`
instead of StateDir on Windows. This plan embeds + installs the sidecar, fixes the
API, switches IPC to loopback-TCP+token, de-stubs CodeShield, REMOVES AlignmentCheck,
bootstraps a venv + gated 22M model with a pinned HF cache, fixes the StateDir bug,
and adds a gated real-sidecar e2e test.

Purpose: a security feature that silently allows everything is worse than none. This
makes the injection scan real and fail-closed, and removes the cloud-exfil path.

Output: embedded+installable sidecar; fixed scan API; TCP+token IPC; real CodeShield;
no AlignmentCheck; venv/model/cache bootstrap; StateDir fix; e2e + honesty edits.

Wave/ordering note: Tier 2 is logically independent of Tier 1, but this plan shares
`internal/config/config.go`, `internal/config/layered.go`, and `cmd/beekeeper/main.go`
with plan 20-01 (Tier 1). To avoid same-wave file conflicts it runs in wave 2 AFTER
20-01 (depends_on: 20-01). Its symbol-level edits (remove AlignmentCheck; StateDir fix
on the llamafirewall handlers) are disjoint from 20-01's (add CatalogSyncConfig;
catalogs daemon handlers) — sequence, not contention.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| agent tool output -> sidecar scan | Untrusted (possibly injection-laden) content crosses into the scanner. |
| Go supervisor <-> Python sidecar (loopback TCP) | Local IPC; any local process could connect without the token. |
| sidecar -> third-party cloud | AlignmentCheck would have exfiltrated agent context to Together AI. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-20-05 | Information Disclosure | AlignmentCheck -> Together AI cloud | mitigate | Remove AlignmentCheck entirely (D-T2-alignment) — no agent context leaves the host; local-first restored. |
