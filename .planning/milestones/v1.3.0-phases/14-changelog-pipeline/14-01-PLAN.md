<design_decisions>
Research was deliberately skipped for this phase. The four open decisions are resolved here and are binding for 14-01 and 14-02:

1. **Architecture — second fumadocs-mdx collection (chosen over a custom per-version route).**
   Add `export const changelog = defineDocs({ dir: 'content/changelog' })` to `web/source.config.ts` ALONGSIDE the existing `export const docs = ...` (do NOT remove or rename `docs`). `web/lib/changelog-source.ts` imports `changelog` from `collections/server` and calls `loader({ baseUrl: '/changelog', source: changelog.toFumadocsSource() })`. The route is the catch-all `web/app/changelog/[[...slug]]/page.tsx` (identical shape to the docs page: `generateStaticParams` returns `source.generateParams()`, Next-16 async `params: Promise<{ slug?: string[] }>`). Each version is its own directory: `content/changelog/v1.0.0/index.mdx`, `content/changelog/v1.2.0/index.mdx`, `content/changelog/v1.3.0/index.mdx` (v1.3.0 authored in 14-02). With `trailingSlash:true`, `generateStaticParams` enumerates exactly the three version slugs and each emits its OWN directory `out/changelog/vX.Y.Z/index.html` (SC-1). Rationale: replicates the empirically-proven Phase 13 pipeline (13-02-SUMMARY) with zero new code patterns and zero new deps; per-version isolation is automatic from the directory layout.

2. **Content scope vs Phase 18 — author real, accurate notes NOW.**
   Phase 14 writes substantive, accurate release notes for the three versions sourced from `.planning/MILESTONES.md` (v1.0.0 + v1.2.0 sections) and the v1.3.0 seed history (Phase 10 + the design-system/docs phases). NOT lorem/placeholder. Phase 18's "accuracy review gate" (DOCS-09) is a final cross-content pass over ALL content, not a reason to defer changelog prose. Every security/verification claim must be accurate to the shipped binary.

3. **GitHub Release links — canonical, honestly framed.**
   Render `https://github.com/bantuson/beekeeper/releases/tag/vX.Y.Z` (lowercase `bantuson` — the go-module / repo path). The repo is not yet pushed (created in the parked v1.1.0 runbook). The ReleaseLinks component frames the link honestly: it is the canonical release URL that resolves once the repo is published — do NOT claim it is live-verified. Do not fabricate asset filenames as live download links beyond what the release will canonically contain (checksums.txt + cosign bundle + SLSA provenance + SBOM).

4. **Nav integration — reachable from docs.**
   Add a Changelog link into the docs `DocsLayout` nav/sidebar (`web/app/docs/layout.tsx`) and give the changelog layout a nav back to docs. Marketing-home nav is Phase 15 scope.

**cosign / SLSA command accuracy (CRITICAL — security product):** The verification commands MUST match the project's release-runbook form. cosign: `cosign verify-blob --bundle checksums.txt.sigstore.json --certificate-identity-regexp '^https://github\.com/Bantuson/beekeeper/' --certificate-oidc-issuer 'https://token.actions.githubusercontent.com' checksums.txt`. **The `--certificate-identity-regexp` MUST use capital-B `Bantuson`** (GitHub OIDC binds to canonical account casing, NOT the lowercase go-module path) — this is Pitfall 4 from docs/release-runbook.md and a wrong command is a security-integrity defect. SLSA: `slsa-verifier verify-artifact --provenance-path <name>.intoto.jsonl --source-uri github.com/Bantuson/beekeeper <artifact>` (capital-B source-uri to match OIDC). SBOM is a CycloneDX `*.cdx.json` asset (mention, no command). Do not invent flags not in the runbook.
</design_decisions>

| Boundary | Description |
