| Boundary | Description |
|----------|-------------|
| committed asset → public CDN | `app/opengraph-image.png` is served verbatim from out/ once deployed; it is hand-authored, not fetched. |
| frontmatter → emitted meta | Page title/description come from MDX frontmatter (trusted, repo-authored). |
| metadataBase constant → canonical/og URLs | All absolute URLs derive from the locked BASE_URL (17-01). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-01 | Tampering / Spoofing | `web/app/opengraph-image.png` (the shared OG card) | mitigate | The card is a hand-authored static asset committed to the repo and served verbatim — NO runtime fetch, NO Edge/ImageResponse generation, NO untrusted/network-sourced image. Supply-chain note: authored locally (or by a committed one-off script), not downloaded. The IHDR 1200×630 check confirms it is the intended file. |
| T-17-04 | Information Disclosure | Home/docs/changelog meta descriptions | mitigate | Per D-04 descriptions state purpose/capabilities only and MUST NOT claim the product is live/installable today (repo unpushed, SITE-03 deferred). Reuse the Phase-14/15 honest framing. |
| T-17-03 | Spoofing | canonical/og:url host | accept | All URLs derive from the hardcoded BASE_URL constant (17-01) — not user input; no host-injection/open-redirect surface. |
| T-17-SC | Tampering | npm/pip/cargo installs | accept | NO new packages installed (uses already-installed next@16.2.7; PNG is a static asset; IHDR check is stdlib struct). No package-legitimacy checkpoint needed — see 17-RESEARCH Package Legitimacy Audit. |
