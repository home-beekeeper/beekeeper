# Phase 21 Plan 03: Sentry Fuzz + CI Matrix Delta Summary

**The 5th and final fuzz target (FuzzEvaluateEvent over the pure Sentry rule evaluator) wired as a blocking CI release gate, plus a 3xGOOS cross-build step — the ci.yml delta added without re-authoring any existing matrix job.**

## Performance

- **Duration:** ~20 min
- **Tasks:** 2
- **Files created:** 1 (fuzz target); **modified:** 1 (ci.yml)
- **Production files modified:** 0 (evaluator is pure, unchanged — D-03)

## Accomplishments
- **VAL-04 complete** — `FuzzEvaluateEvent` (`//go:build fuzz`) fuzzes the pure `sentry.EvaluateEvent` entry point. Contract: never panics + only ever returns `critical`/`high` severities. 5s local smoke ran 22.9k execs with no crash. This is the 5th required fuzz target (policy, IPC-proto, catalog, MCP=gateway/parser, **Sentry**).
- **VAL-03 CI delta** — `ci.yml` gains: (1) a Linux-leg "Cross-compile (3 GOOS, build-only)" step gating cross-compilation; (2) a new `fuzz-sentry` job (`-fuzztime 30s`); (3) `fuzz-sentry` added to `release-gate.needs` so a discovered Sentry panic blocks release (VAL-08). The pre-existing eBPF-generate, kernel-5.4/5.15 little-vm-helper, eslogger, and `-race`/CGO jobs are all preserved.
- **Isolation intact** — `web.yml`/`release.yml` untouched; `paths-ignore: web/**` preserved on both ci.yml triggers (Pitfall 4).

## Task Commits
1. **Task 1: FuzzEvaluateEvent** — `test(21-03)` (Sentry rule-evaluator fuzz target)
2. **Task 2: ci.yml matrix delta** — `ci(21-03)` (3xGOOS cross-build + fuzz-sentry blocking gate)

## Decisions Made
- **Cross-compile gated to Linux.** The eBPF bytecode is generated only on the Linux leg; running the 3-GOOS build there means `GOOS=linux` uses the real generated `sentry/linux` package while `GOOS=darwin/windows` build their platform files. Running it on the macOS/Windows legs would try to cross-build the linux eBPF package without generated bytecode. One run covers all 3 targets, so this is both safer and sufficient.
- **No new ipc CI job.** The 21-01 ipc Tier-B tests (`//go:build linux||darwin`) are picked up by the existing matrix `go test -race ./...` on the Linux + macOS legs — confirmed `./...` includes `./internal/ipc/...`.

## Deviations from Plan
- **Cross-compile step is Linux-gated** (RESEARCH's illustrative YAML showed an ungated step). Gating to Linux avoids the macOS/Windows runners cross-building the eBPF package without bytecode, and is still a complete 3-GOOS gate. No functional loss.

## Issues Encountered
None — the fuzz target's `BaselineState{}` zero-value was safe to pass (RESEARCH A1 confirmed); the seed corpus and 5s smoke found no crashes.
