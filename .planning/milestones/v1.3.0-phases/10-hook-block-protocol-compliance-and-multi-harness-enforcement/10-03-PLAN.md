<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| installer → harness config file | The installer writes the event names + command + feature flags the harness reads; a wrong event name (the Cursor bug) means the hook never fires → silent fail-open |
| installer → existing user hooks | The installer must merge, never clobber, a user's pre-existing hooks |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-09 | Tampering (hook never fires) | Cursor preToolUse bug | mitigate | The Cursor test asserts preToolUse is ABSENT and the three real events (beforeShellExecution/MCPExecution/ReadFile) are present with the --hook command — the exact misconfiguration that silently disabled the hook can no longer ship |
| T-10-10 | Tampering (Cursor fail-open) | Cursor failClosed | mitigate | The cursorHook sets FailClosed:true and the test asserts it; Cursor defaults to fail-OPEN, so a crash/timeout without this would allow the tool |
| T-10-11 | Tampering (Codex hook disabled) | Codex [features] flag | mitigate | ensureCodexFeaturesFlag writes [features] hooks=true (PR #18385 gate) idempotently; the test asserts the flag is present — without it Codex ignores the hook entirely |
| T-10-12 | Spoofing (clobber user hooks) | Augment/CodeBuddy/Qwen install | mitigate | All three reuse mergeClaudeHookEntry + editorinit.PatchSettings (sets only the hooks key); the preserves_existing_hooks subtest seeds a foreign hook and asserts survival |
| T-10-SC | Tampering | npm/pip/cargo installs | accept | No package installs and NO new module dependency — the Codex TOML patch is a careful in-repo string edit, not a new TOML library (CLAUDE.md: no new deps) |
</threat_model>
