Build the Windows ingestion layer: add tekert/golang-etw to go.mod, define provider GUIDs, build the ETW consumer wrapper that pumps events into the platform-agnostic SentryEvent channel, normalize Kernel-Process / Kernel-File / Kernel-Network ETW events into SentryEvent, and probe NT Kernel Logger session conflicts. This plan covers SWIN-02 (ETW event ingestion via tekert/golang-etw — no CGO), SWIN-03 (NT Kernel Logger conflict probe), and SWIN-04 (EventsLost counter surfaced for beekeeper diag).

    `func parseETWEvent(e *etw.Event) (sentry.SentryEvent, error)`:
    - guid := normalizeGUID(e.System.Provider.Guid).
    - Defensive: if e is nil OR e.System.Execution.ProcessID == 0 (System Idle Process) AND eventID >= some threshold, return ErrUnknownEvent. Specifically: drop System Idle Process (PID 0) for any event because it pollutes the rule engine.
    - wallTime := e.System.TimeCreated.SystemTime; if wallTime.IsZero() { wallTime = time.Now().UTC() }.
    - Switch on guid:

      Case guidKernelProcess (Process events, Event IDs per Microsoft-Windows-Kernel-Process):
        Per Microsoft public docs, Event ID 1 = ProcessStart, Event ID 2 = ProcessStop. Map ID 1 to EventProcessCreate.
        - eventID := e.System.EventID; if eventID != 1 return ErrUnknownEvent (skip stops and other IDs).
        - imageName, _ := e.EventData["ImageName"].(string)  // or "ImagePath" / "ImageFileName" — exact key depends on tekert/golang-etw event property naming
        - cmdline, _ := e.EventData["CommandLine"].(string)
        - parentPIDraw := e.EventData["ParentProcessID"]; parse to uint32 with a helper toUint32 that handles both float64 (JSON-decoded) and uint32.
        - Return SentryEvent{ Kind: sentry.EventProcessCreate, PID: e.System.Execution.ProcessID, PPID: parentPID, Exe: imageName, Cmdline: cmdline, WallTime: wallTime }, nil.

      Case guidKernelFile:
        - File create / open event IDs vary; accept the union {12, 14, 15} (Create, NameCreate, NameDelete) and map all to EventFileAccess. If event ID is outside that set return ErrUnknownEvent.
        - filePath, _ := e.EventData["FileName"].(string); if filePath == "" filePath, _ = e.EventData["FilePath"].(string).
        - Return SentryEvent{ Kind: sentry.EventFileAccess, PID: e.System.Execution.ProcessID, FilePath: filePath, WallTime: wallTime }, nil.

      Case guidKernelNetwork:
        - TcpIp/Connect typically Event ID 12 or "TcpIpConnect" task-name. Accept Event IDs {10, 11, 12} (any TcpIp start variant); other IDs → ErrUnknownEvent.
        - daddrStr, _ := e.EventData["daddr"].(string); if daddrStr == "" daddrStr, _ = e.EventData["DestinationAddress"].(string).
        - portRaw := e.EventData["dport"]; if portRaw == nil portRaw = e.EventData["DestinationPort"]; port := toUint16(portRaw).
        - Return SentryEvent{ Kind: sentry.EventNetworkConnect, PID: e.System.Execution.ProcessID, DstAddr: net.ParseIP(daddrStr), DstPort: port, WallTime: wallTime }, nil.

      Default: return zero, fmt.Errorf("%w: provider %s", ErrUnknownEvent, guid).

    Helper `func toUint32(v interface{}) uint32`: switch v.(type) { case uint32: return v.(uint32); case uint64: return uint32(v.(uint64)); case float64: return uint32(v.(float64)); case int: return uint32(v.(int)); case string: parse with strconv.ParseUint; default: return 0 }.

    Helper `func toUint16(v interface{}) uint16`: same shape but to uint16.
