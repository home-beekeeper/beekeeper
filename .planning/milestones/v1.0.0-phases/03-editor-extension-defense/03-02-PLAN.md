<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| marketplace API → adapter | Untrusted JSON (timestamp string) crosses from a remote registry into the age computation |
| extension manifest fields → quarantine path | Attacker-controlled publisher/name/version used to compose a filesystem path |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-02-01 | Tampering | Quarantine path traversal via `publisher`/`name`/`version` = `../../etc` | mitigate | filepath.Base() on every attacker-controlled segment before composing the id; filepath.Clean + strings.HasPrefix prefix-check that destDir stays under ExtensionsDir; reject otherwise. Restore/Purge apply filepath.Base on the supplied id. |
| T-03-02-02 | Tampering | Marketplace timestamp poisoning via MITM | mitigate | HTTPS-only base URLs; 24h TTL cache; both-API-failure → Missing:true → release-age policy blocks (fail-closed). Negative/future age treated as missing. |
| T-03-02-03 | Information Disclosure | beekeeper-manifest.json world-readable (contains original install path) | mitigate | os.WriteFile 0o600 + platform.SetOwnerOnly on the manifest, matching the audit-log ownership pattern. |
| T-03-02-04 | Denial of Service | Oversized marketplace JSON response | mitigate | 4MB io.LimitReader cap on both Open VSX (via fetchRegistryJSON) and the manual VS Code Marketplace POST. |
| T-03-02-05 | Tampering | Cross-volume rename silently corrupting quarantine on Windows | accept | Documented as unsupported in Phase 3 (Pitfall 5); Move returns a wrapped error on cross-device link rather than a partial move. Quarantine dir on same drive as %APPDATA% is the supported configuration. |
</threat_model>
