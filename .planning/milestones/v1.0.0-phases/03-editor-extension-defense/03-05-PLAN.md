<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| editor settings.json → patcher | An existing user-authored (possibly JSONC) file is read and rewritten |
| PATH/filesystem → detector | Editor presence is probed but never trusted to mutate beekeeper state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05-01 | Tampering | PatchSettings corrupting a user's settings.json (partial write) | mitigate | Atomic write: marshal fully, write to a temp file in the same dir, os.Rename into place; never truncate-in-place. Parse failure on the existing file returns an error BEFORE any write. |
| T-03-05-02 | Information Disclosure | Detection probing leaking which editors are installed | accept | Detection is local-only (exec.LookPath + os.Stat); results stay in-process and inform the consent prompt; no exfiltration surface. |
| T-03-05-03 | Tampering | Comment loss silently altering user intent in settings.json | mitigate | Documented limitation in code + the plan-04 consent prompt warns the user explicitly before writing (Approach A). Idempotent patch avoids repeated churn. |
| T-03-05-04 | Spoofing | A malicious entry slipping past EDXT-01 recognition undetected by the binary | mitigate | The selftest corpus fixture makes EDXT-01 recognition a release-verifiable behavior: beekeeper selftest fails if code --install-extension no longer routes a known-malicious id through the catalog. |
</threat_model>
