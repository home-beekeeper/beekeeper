Backup pattern (from RESEARCH code example):
```go
func backupSettings(path string) error {
    data, err := os.ReadFile(path)
    if err != nil {
        if errors.Is(err, os.ErrNotExist) { return nil }
        return err
    }
    ts := time.Now().Format("20060102-150405")
    backupPath := path + ".beekeeper-backup-" + ts
    return os.WriteFile(backupPath, data, 0o644)
}
```

Atomic write helper (follow catalog/state.go writeFileAtomic pattern):
```go
func writeFileAtomic(path string, data []byte) error {
    tmp := path + ".tmp"
    if err := os.WriteFile(tmp, data, 0o644); err != nil { return err }
    return os.Rename(tmp, path)
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| --target flag → file path selection | The target is an enum; no user-supplied file path accepted |
| installer → settings.json / hooks.json | File write crosses into editor configuration; backup guards against data loss |
| hooks.json content → agent hook runner | Hook commands are hardcoded ("beekeeper check"); not user-supplied |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01-01 | Tampering | Hook installer writing to wrong file via target flag path traversal | mitigate | --target is a closed enum (claude-code, cursor, codex, continue, opencode, openclaw); file paths are derived from os.UserHomeDir() internally, never from user input |
| T-04-01-02 | Tampering | installCursor writing hooks to ~/.cursor/settings.json instead of hooks.json | mitigate | Cursor installer writes exclusively to ~/.cursor/hooks.json per RESEARCH Pattern 2; settings.json path is never constructed in cursor.go (verified by acceptance criteria grep) |
| T-04-01-03 | Tampering | Hook installer overwriting settings.json without backup | mitigate | backupSettings is called before any write; missing file case returns nil (not an error) |
| T-04-01-04 | Elevation of Privilege | Cursor failClosed:false allowing hook crash = fail-open | mitigate | failClosed:true is hardcoded in cursor.go; acceptance criteria grep verifies its presence |
| T-04-01-05 | Denial of Service | Idempotency failure duplicating hooks entries causing agent slowdown | mitigate | containsHookByCommand guard before append in all three installers; TestInstallCursor/TestInstallCodex verify no duplication |
| T-04-01-06 | Information Disclosure | Gateway guide printing stale or wrong token | mitigate | printGatewayGuide prints instructions to run "beekeeper gateway token" rather than a token value; token is never embedded in printed text |
</threat_model>
