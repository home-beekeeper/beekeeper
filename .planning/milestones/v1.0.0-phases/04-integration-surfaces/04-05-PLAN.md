<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| beekeeper gateway --upstream flag → upstream URL | User-supplied URL; gateway must not SSRF to unintended hosts |
| tool output → audit record | Sensitive credentials may appear in tool outputs before redaction |
| beekeeper gateway token → stdout | Token printed to terminal; must be masked in status command |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05-01 | Information Disclosure | Gateway token printed in full to terminal via status | mitigate | gateway status masks token to last 8 chars + "..."; gateway token prints full token (user explicitly requested it) |
| T-04-05-02 | Information Disclosure | API keys / JWT tokens in audit log (tool outputs) | mitigate | RedactRecord applied in writeAudit before writing; TestRedactBearerToken and TestRedactJWT verify |
| T-04-05-03 | Tampering | --upstream flag pointing to internal service (SSRF) | accept | Gateway is a localhost tool for developer use; upstream URL is user-configured; no SSRF mitigation in Phase 4 (the upstream is the user's own MCP server). Document in gateway --help. |
| T-04-05-04 | Denial of Service | beekeeper audit-record called with large stdin causing OOM | mitigate | RunAuditRecord uses the same maxStdin 1MB cap as RunCheck (inherited from check package); oversized input is truncated/ignored; still exits 0 |
| T-04-05-05 | Elevation of Privilege | beekeeper hooks install --target with a crafted target bypassing enum check | mitigate | hooks.Install validates target against supported list; returns non-nil error for unknown targets; TestInstallUnknownTarget in Plan 01 covers this |
| T-04-05-06 | Repudiation | Gateway state cleanup not happening on crash | accept | Gateway crash without graceful shutdown leaves stale PID in state.json; gateway status shows stale state; next gateway start overwrites with fresh token. Documented behavior. |
| T-04-05-07 | Tampering | Regex-based redaction with catastrophic backtracking | mitigate | Use simple non-backtracking patterns (character classes, not nested quantifiers); test each pattern with pathological inputs in redact_test.go |
</threat_model>
