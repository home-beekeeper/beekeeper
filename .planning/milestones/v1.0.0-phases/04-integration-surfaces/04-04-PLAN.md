<objective>
Implement the shim layer (internal/shim/) — OS-native wrapper scripts placed in ~/.beekeeper/shims/ that intercept package manager and toolchain invocations by invoking beekeeper check before forwarding to the real binary.

Purpose: INTG-06. PATH-prepended shims provide universal coverage for any agent that executes package managers directly (npm, pip, cargo, etc.) without going through a hook or MCP gateway. This completes the defense-in-depth story: native hooks cover hook-enabled agents, the MCP gateway covers MCP clients, and shims cover everything else. The cmd/beekeeper/main.go Cobra wiring lives in Plan 05; this plan owns all shim business logic.

Output: internal/shim/ package with Install/Uninstall/Status functions; platform-specific shim writers for Unix and Windows; full test suite.

Unix shim script template (RESEARCH Pattern 9 — VERIFIED):
```sh
#!/bin/sh
# beekeeper shim for npm — auto-generated, do not edit
# Real binary: /usr/local/bin/npm
beekeeper check <<EOF
{"tool_name":"Bash","tool_input":{"command":"npm $*"}}
EOF
_bk_exit=$?
if [ $_bk_exit -eq 0 ]; then
    exec "/usr/local/bin/npm" "$@"
fi
exit $_bk_exit
```
CRITICAL: `exec` replaces the shell to preserve signal handling and exit codes.
File written to: ~/.beekeeper/shims/npm (no extension), chmod 0755.

Windows .cmd batch file template (RESEARCH Pattern 9 — VERIFIED):
```cmd
@echo off\r\n
echo {"tool_name":"Bash","tool_input":{"command":"npm %*"}} | beekeeper check\r\n
if %ERRORLEVEL% EQU 0 goto :run\r\n
exit /b %ERRORLEVEL%\r\n
:run\r\n
"C:\Program Files\nodejs\npm.cmd" %*\r\n
```
CRITICAL: CRLF (\r\n) line endings required for Windows cmd.exe. Quote the real binary path.
File written to: ~/.beekeeper/shims/npm.cmd

findRealBinary pattern (RESEARCH Pattern 9):
```go
func findRealBinary(shimDir, tool string) (string, error) {
    origPath := os.Getenv("PATH")
    filteredPath := filterPathEntries(origPath, shimDir)
    os.Setenv("PATH", filteredPath)
    defer os.Setenv("PATH", origPath)
    return exec.LookPath(tool)
}

func filterPathEntries(path, exclude string) string {
    sep := string(os.PathListSeparator)
    parts := strings.Split(path, sep)
    var filtered []string
    for _, p := range parts {
        if filepath.Clean(p) != filepath.Clean(exclude) {
            filtered = append(filtered, p)
        }
    }
    return strings.Join(filtered, sep)
}
```
Safe for CLI (single goroutine). Add note: not goroutine-safe; if called from concurrent context, use a sync.Mutex.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| PATH resolution → real binary path | findRealBinary must never resolve to a shim to prevent infinite loops |
| shim script → beekeeper check stdin | Tool name and args from user flow through to beekeeper check |
| real binary path embedded in shim | Must be the actual tool binary, not attacker-controlled |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04-01 | Tampering | Shim pointing to itself (infinite loop when npm called) | mitigate | filterPathEntries removes shimDir from PATH before LookPath; TestShimRealBinary verifies; also write note in shim script header that modifying it is unsupported |
| T-04-04-02 | Tampering | Real binary path with spaces breaking .cmd execution | mitigate | Windows shim template quotes the real binary path in double quotes; acceptance criteria grep verifies quoted path in shim content |
| T-04-04-03 | Denial of Service | Shim left in place after tool uninstall → beekeeper check blocks a missing binary | mitigate | shim uninstall removes all shims; shim status shows broken shims; users informed via PATH instructions to run shim install to refresh |
| T-04-04-04 | Tampering | OS.Setenv PATH modification during concurrent goroutines | accept | findRealBinary is only called from shim install (single-goroutine CLI command); add code comment that this is not goroutine-safe; Pitfall 5 documented in RESEARCH |
| T-04-04-05 | Tampering | Windows .cmd CRLF requirement not met → silent exec failure | mitigate | shim_windows.go uses "\r\n" throughout; TestShimInstallWindows verifies CRLF bytes present |
| T-04-04-06 | Tampering | Tool args injection via $* or %* passing malicious arguments to real binary | accept | Shims are intended to proxy all args to the real binary after policy allow; arg content is evaluated by beekeeper check before proxy; args passing is correct behavior |
</threat_model>
