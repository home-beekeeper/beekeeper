<objective>
Extend the policy engine and audit record with multi-agent context propagation (INTG-07). This adds the AgentContext pure struct to internal/policy/types.go, extends the Evaluate signature to enforce agent depth limits, adds lineage fields to AuditRecord, and wires env-var reading into the check handler.

Purpose: Subagent tool calls must carry parent-child lineage so the policy engine can enforce that child agents cannot exceed parent policy permissions, and so the audit log provides a forensic parent chain for every blocked or warned tool call. The beekeeper audit-record command (PostToolUse) is also added here since it is required by the Claude Code hook format written in Plan 01.

Output: AgentContext struct; extended Evaluate; lineage fields in AuditRecord; readAgentContext in handler; beekeeper audit-record Cobra command handler function (cmd is registered in Plan 05 main.go, but the runAuditRecord function lives here in internal/check/ so Plan 05 can import it).
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| BEEKEEPER_AGENT_DEPTH env var → depth integer | Attacker-controlled env var crosses into depth enforcement logic |
| BEEKEEPER_AGENT_LINEAGE env var → string slice | Comma-separated parent IDs cross into audit record |
| Claude Code hook stdin agent_id → AgentContext | Untrusted JSON field crosses into policy context |
| PostToolUse stdin → audit record | Unvalidated hook output crosses into the audit log |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-02-01 | Elevation of Privilege | BEEKEEPER_AGENT_DEPTH=-1 bypassing depth limit | mitigate | Negative depth normalized to 0 (root) in readAgentContext; TestReadAgentContextNegativeDepth verifies; policy engine also normalizes before depth check |
| T-04-02-02 | Elevation of Privilege | Agent depth > maxAgentDepth reaching policy engine unchecked | mitigate | Depth check is the FIRST operation in Evaluate, before any corroboration logic; returns block immediately |
| T-04-02-03 | Tampering | Comma-containing parent IDs corrupting lineage split | accept | A8 in RESEARCH assumptions log: known limitation; IDs from env are assumed not to contain commas; documented for future revision (URL-encode or base64 in a later phase) |
| T-04-02-04 | Repudiation | PostToolUse audit-record failure silently not recorded | mitigate | RunAuditRecord logs write failures to stderr but still exits 0; the PostToolUse hook must not disrupt the agent |
| T-04-02-05 | Spoofing | Malicious Claude Code stdin injecting arbitrary agent_id | accept | agent_id is stored in the audit log for forensics only; it does not change the corroboration decision; env var overrides stdin agent_id, so orchestration layer has final say |
| T-04-02-06 | Tampering | AgentContext polluting policy engine with I/O (env reads in policy package) | mitigate | readAgentContext lives exclusively in internal/check/handler.go (I/O tier); policy.AgentContext is a pure value struct with no methods or I/O; verified by architecture constraint (internal/policy must be pure) |
