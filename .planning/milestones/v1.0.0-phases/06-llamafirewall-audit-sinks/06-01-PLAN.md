    Create internal/audit/rotate.go (no build tag — all platforms):

    Package: `package audit`

    Imports: `encoding/json`, `fmt`, `os`, `path/filepath`, `time`, plus `github.com/mzansi-agentive/beekeeper/internal/platform`.

    Implement `func Rotate(auditPath string, maxBytes int64, retentionDays int) error`:

    1. Stat auditPath. If os.ErrNotExist, return nil (nothing to rotate). If size <= maxBytes, return nil (no-op).
    2. Find existing numbered archives: enumerate beekeeper.ndjson.N files in filepath.Dir(auditPath) by iterating N from 1 upward until os.Stat returns ErrNotExist. Cap scan at 999 to prevent infinite loop.
    3. Delete archives older than retentionDays: for each numbered archive, check its mtime via os.Stat. If time.Since(info.ModTime()) > time.Duration(retentionDays)*24*time.Hour, os.Remove it. Continue scanning all archives (do not stop at first non-old).
    4. Shift surviving archives up: rename beekeeper.ndjson.N → beekeeper.ndjson.(N+1) in reverse order (highest N first) to avoid clobbering. Use os.Rename.
    5. Rename beekeeper.ndjson → beekeeper.ndjson.1.
    6. Create new beekeeper.ndjson: os.OpenFile(auditPath, os.O_CREATE|os.O_WRONLY|os.O_TRUNC, 0600). Close immediately after create.
    7. Call platform.SetOwnerOnly(auditPath) on the new file.
    8. Return any error from steps 3-7 wrapped with fmt.Errorf.

    Note: Rotate is NOT concurrent-safe on its own. The Writer mutex (Phase 6 Writer extension) serializes calls. Rotate itself has no mutex.

    Create internal/audit/rotate_test.go (no build tag):

    TestRotateCreatesNumberedArchive: write 100 bytes to a temp auditPath, call Rotate with maxBytes=50. Assert beekeeper.ndjson.1 exists and beekeeper.ndjson exists with size 0. Check perms on Unix (skip perm check on Windows via runtime.GOOS).

    TestRotateShiftsExistingArchives: create beekeeper.ndjson (100 bytes) and beekeeper.ndjson.1 (5 bytes). Call Rotate(maxBytes=50). Assert beekeeper.ndjson.2 exists (was .1), beekeeper.ndjson.1 exists (was current), beekeeper.ndjson exists and is empty.

    TestRotateDeletesOldArchives: create beekeeper.ndjson.1 in a tempdir, set its mtime to 40 days ago via os.Chtimes. Create beekeeper.ndjson (100 bytes). Call Rotate(maxBytes=50, retentionDays=30). Assert beekeeper.ndjson.1 does NOT exist (old archive deleted) and beekeeper.ndjson.1 (the rotated current) exists.

    TestRotateNoOpWhenSmall: write 10 bytes to auditPath, call Rotate(maxBytes=1000). Assert no beekeeper.ndjson.1 exists and beekeeper.ndjson still has 10 bytes.

    type QueryOpts struct {

    Implement `func Query(ctx context.Context, r io.Reader, opts QueryOpts, out io.Writer) error`:
    - Use bufio.Scanner with a 1MB buffer (scanner.Buffer(make([]byte, 1<<20), 1<<20)) to handle long NDJSON lines.
    - For each line: attempt json.Unmarshal into AuditRecord. On unmarshal error, increment a skipped counter and continue.
    - Apply filters in order: Since (skip if rec.Timestamp < opts.Since after time.Parse(time.RFC3339, rec.Timestamp); skip parse errors as malformed), Agent (skip if opts.Agent != "" && rec.AgentName != opts.Agent), Tool (skip if opts.Tool != "" && rec.ToolName != opts.Tool), Decision (skip if opts.Decision != "" && rec.Decision != opts.Decision).
    - Write matching record as the original raw line bytes + newline to out (do NOT re-marshal; preserve original JSON shape).
    - Check ctx.Err() every 100 lines; return ctx.Err() if cancelled.
    - Apply Limit: if opts.Limit > 0 and written >= opts.Limit, stop scanning.
    - After loop: if skipped > 0, fmt.Fprintf(out, "# %d malformed line(s) skipped\n", skipped). Return nil.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| NDJSON file → Query/Export reader | Malformed lines from external file modification must not crash or leak memory |
| CLI --since/--agent/--tool flags → Query filter | User-supplied filter values are matched by string equality only; no regex expansion |
| Export OTLP output → external collector | OTLP JSON is hand-rolled; no credentials or secrets should appear in body.stringValue after redaction was applied at write time |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01-01 | Denial of Service | Query/Export bufio.Scanner on large audit file | mitigate | 1MB scanner buffer cap; streaming (no full-file slurp); Limit flag caps output count |
| T-06-01-02 | Information Disclosure | audit export --format otlp sending audit data off-machine | accept | OTLP export writes to stdout only; OTLP sink (Plan 03) has the network path with documented warning |
| T-06-01-03 | Tampering | Rotate race with concurrent Writer.Write | mitigate | Rotate is called under Writer mutex (Phase 6 Writer extension per 06-CONTEXT.md); Rotate itself has no mutex as documented |
| T-06-01-04 | Information Disclosure | New beekeeper.ndjson created with wrong permissions after rotation | mitigate | platform.SetOwnerOnly called immediately after os.OpenFile in Rotate; same pattern as Writer.NewWriter |
| T-06-01-05 | Denial of Service | OTLP export collecting unbounded records into memory | mitigate | 100,000 record cap on OTLP collection; streaming CSV/NDJSON formats have no in-memory accumulation |
</threat_model>
