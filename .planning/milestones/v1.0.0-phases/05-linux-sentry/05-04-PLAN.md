- Tier 0 (>=5.15): FAN_CLASS_CONTENT|FAN_NONBLOCK|FAN_CLOEXEC|FAN_REPORT_FID|FAN_REPORT_PIDFD

    Implement `func StartFanotifyReader(ctx context.Context, fanFd int, events chan<- sentry.SentryEvent)`:
    This is a goroutine entrypoint (called with `go StartFanotifyReader(...)`).

    Use an epoll fd to avoid spinning on EAGAIN:
    - epollFd, _ := unix.EpollCreate1(unix.EPOLL_CLOEXEC)
    - unix.EpollCtl(epollFd, unix.EPOLL_CTL_ADD, fanFd, &unix.EpollEvent{Events: unix.EPOLLIN, Fd: int32(fanFd)})
    - Cancel goroutine: `go func() { <-ctx.Done(); unix.Close(epollFd) }()`
    - Main loop: epoll_wait with timeout 500ms; on ctx.Done() break; on fanFd EPOLLIN: read events

    Event reading inner loop (called from above when EPOLLIN fires):
    ```
    buf := make([]byte, 4096)
    n, err := unix.Read(fanFd, buf)
    ```
    Parse buffer: offset := 0; while offset < n:
    - Cast to *unix.FanotifyEventMetadata using `unsafe.Pointer(&buf[offset])`
    - Guard: if meta.Event_len < unix.FAN_EVENT_METADATA_LEN { break }
    - Resolve path: if meta.Fd >= 0: `path, _ = os.Readlink(fmt.Sprintf("/proc/self/fd/%d", meta.Fd))`
    - CRITICAL: `unix.Close(int(meta.Fd))` immediately after Readlink (RESEARCH Pitfall 3)
    - Respond to FAN_OPEN_PERM: if meta.Mask & unix.FAN_OPEN_PERM != 0: write FAN_ALLOW response to fanFd:
      ```go
      resp := [8]byte{}
      binary.LittleEndian.PutUint32(resp[0:4], uint32(meta.Fd))  // Fd field (already closed but response uses original value)
      binary.LittleEndian.PutUint32(resp[4:8], unix.FAN_ALLOW)
      unix.Write(fanFd, resp[:])
      ```
      NOTE: The Fd in the response must be the ORIGINAL meta.Fd value before Close. Read meta.Fd into a local variable BEFORE calling unix.Close.
    - Emit SentryEvent: non-blocking send: `select { case events <- sentry.SentryEvent{Kind: sentry.EventFileAccess, PID: uint32(meta.Pid), FilePath: path, WallTime: time.Now().UTC()}: default: }` — drop if channel full; fd already closed and FAN_ALLOW already sent, so the accessing process is never blocked
    - offset += int(meta.Event_len)

    Implement `func DropCapabilities(keepCaps []uintptr) error`:

    CRITICAL: Use `var data [2]unix.CapUserData` (array of two, NOT single struct) — golang/go#44312 fix.

    ```go
    func DropCapabilities(keepCaps []uintptr) error {
        hdr := unix.CapUserHeader{
            Version: unix.LINUX_CAPABILITY_VERSION_3,
            Pid:     0, // 0 = current process
        }
        var data [2]unix.CapUserData

        if err := unix.Capget(&hdr, &data[0]); err != nil {
            return fmt.Errorf("capget: %w", err)
        }

        // Zero all capabilities
        data[0].Effective   = 0
        data[0].Permitted   = 0
        data[0].Inheritable = 0
        data[1].Effective   = 0
        data[1].Permitted   = 0
        data[1].Inheritable = 0

        // Re-enable only the kept capabilities
        for _, cap := range keepCaps {
            idx := int(cap / 32)
            bit := uint32(1 << (cap % 32))
            if idx == 0 {
                data[0].Effective   |= bit
                data[0].Permitted   |= bit
            } else if idx == 1 {
                data[1].Effective   |= bit
                data[1].Permitted   |= bit
            }
            // idx > 1 would be cap >= 64 — not valid for current Linux; skip
        }

        return unix.Capset(&hdr, &data[0])
    }
    ```

    Implement `func keepCaps(tier DegradationTier) []uintptr` — returns the capability set appropriate for each tier AFTER eBPF objects are loaded:
    - All tiers retain: unix.CAP_NET_ADMIN (needed for fanotify on network sockets), unix.CAP_DAC_READ_SEARCH (needed for /proc/pid/fd/ symlink resolution)
    - No other capabilities are retained after eBPF load (CAP_BPF and CAP_SYS_ADMIN are dropped)

    Implement `func ApplySeccomp() error`:
    Use `github.com/elastic/go-seccomp-bpf` (per RESEARCH §6.2 recommendation and instruction prompt's key architecture decision). Note that CONTEXT.md says "raw unix.Prctl path" but the instruction prompt's Key Architecture Decisions section overrides this with `github.com/elastic/go-seccomp-bpf v1.0.2` (already added to go.mod in 05-03).

    ```go
    func ApplySeccomp() error {
        filter := seccomp.Filter{
            NoNewPrivs: true,
            Flag:       seccomp.FilterFlagTSync,
            Policy: seccomp.Policy{
                // Use allow-list approach with ActionAllow as default is TOO risky for Go runtime.
                // Use ActionAllow default + block known privilege-escalation syscalls.
                DefaultAction: seccomp.ActionAllow,
                Syscalls: []seccomp.SyscallGroup{
                    {
                        Action: seccomp.ActionKillProcess,
                        Names: []string{
                            "execve", "execveat",  // prevent spawning new processes
                            "ptrace",               // prevent process inspection
                            "kexec_load",           // prevent kernel loading
                            "reboot",               // prevent reboot
                            "swapon", "swapoff",    // prevent swap manipulation
                            "mount", "umount2",     // prevent filesystem manipulation
                        },
                    },
                },
            },
        }
        return seccomp.LoadFilter(filter)
    }
    ```

    Rationale for ActionAllow + blocklist (not allowlist): Go runtime uses many syscalls (futex, clone, mmap, epoll_create1, sigaltstack, getrandom, etc.) that vary by Go version and platform. A too-strict allowlist causes SIGSYS crashes (RESEARCH Pitfall 8). The blocklist approach kills on known privilege-escalation syscalls while allowing the runtime to function. This is explicitly documented in RESEARCH §6.2.

    Implement `func sdNotifyReady() error` — hand-rolled NOTIFY_SOCKET writer (per RESEARCH §7.3 10-line alternative, avoids coreos/go-systemd dependency):
    ```go
    func sdNotifyReady() error {
        socketPath := os.Getenv("NOTIFY_SOCKET")
        if socketPath == "" {
            return nil // not running under systemd; not an error
        }
        conn, err := net.Dial("unixgram", socketPath)
        if err != nil {
            return fmt.Errorf("sd_notify dial: %w", err)
        }
        defer conn.Close()
        _, err = conn.Write([]byte("READY=1"))
        return err
    }
    ```
    Add `net` and `os` to imports.

    Note: privilege.go is NOT tested in isolation on Windows — it's all linux build-tagged. The daemon integration test in 05-05 covers it on LVH CI.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| fanotify fd events → SentryEvent | Kernel-generated event bytes; Fd values are kernel-assigned and must be closed promptly |
| DropCapabilities → running process | After call, process must not be able to re-escalate; seccomp applied immediately after |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-04-01 | Denial of Service | FAN_OPEN_PERM event not responded to — accessing process hangs indefinitely | mitigate | FAN_ALLOW response written to fanFd BEFORE emitting to channel; even if channel is full (drop path), FAN_ALLOW is always sent; response uses original meta.Fd value saved before unix.Close |
| T-05-04-02 | Denial of Service | fanotify fd exhaustion from unclosed event fds | mitigate | unix.Close(int(meta.Fd)) called immediately after Readlink, before any channel operation; covered by RESEARCH Pitfall 3 |
| T-05-04-03 | Elevation of Privilege | Capability re-escalation after DropCapabilities via execve | mitigate | seccomp blocks execve and execveat after ApplySeccomp; NoNewPrivs:true in seccomp filter prevents gaining new privileges via setuid |

| T-05-04-05 | Tampering | seccomp filter not applied to all goroutine threads | mitigate | FilterFlagTSync syncs the filter to all threads in the process via SECCOMP_FILTER_FLAG_TSYNC |
