    Create internal/sentry/linux/probe.go with build tag `//go:build linux`:

    Define DegradationTier type and constants:
    ```go
    type DegradationTier int
    const (
        Tier0 DegradationTier = iota // Full: ring buffer + FAN_REPORT_FID
        Tier1                         // Degraded: perf buffer, no FAN_REPORT_FID
        Tier2                         // Minimal: fanotify only, no eBPF
    )
    ```

    Implement `probeTier() DegradationTier`:
    1. Check CAP_BPF/CAP_SYS_ADMIN via unix.Capget with `var data [2]unix.CapUserData` (CRITICAL — two-struct pattern from RESEARCH Pitfall 1 / golang/go#44312):
       ```go
       hdr := unix.CapUserHeader{Version: unix.LINUX_CAPABILITY_VERSION_3, Pid: 0}
       var data [2]unix.CapUserData
       unix.Capget(&hdr, &data[0])
       // Check CAP_BPF (bit 39 = index 1, bit 7) and CAP_SYS_ADMIN (bit 21 = index 0, bit 21)
       capBPF := (data[1].Effective >> (39 % 32)) & 1
       capSysAdmin := (data[0].Effective >> (21 % 32)) & 1
       hasPriv := capBPF != 0 || capSysAdmin != 0
       ```
    2. If !hasPriv: return Tier2
    3. `err := features.HaveMapType(ebpf.RingBuf)` — if errors.Is(err, ebpf.ErrNotSupported): return Tier1

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-03-01 | Tampering | Malicious eBPF event with oversized exe/cmdline fields causing buffer overread | mitigate | Go-side parsing uses binary.Read into fixed-size [256]byte and [512]byte arrays; no variable-length reads; truncation is safe |
| T-05-03-02 | Denial of Service | Ring buffer overflow dropping security-relevant events silently | mitigate | EventsDropped counter incremented atomically; surfaced in beekeeper protect status and beekeeper diag (Phase 9); operator is informed of coverage degradation |

| T-05-03-04 | Elevation of Privilege | eBPF verifier bypass via crafted BPF bytecode | mitigate | Pre-compiled bytecode, never runtime-compiled; kernel eBPF verifier runs at load time and rejects invalid programs; no user-supplied BPF |
