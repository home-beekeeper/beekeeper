<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| state.json on disk → probeLlamaFirewall | The sidecar supervisor writes state.json; a partial, corrupt, or older-schema file is untrusted input that must not crash the dashboard |
| OS process table → PID liveness check | os.FindProcess / signal-0 result drives only the pip color; no action is taken on the process |
| HealthState → base.go pip render | Health booleans affect display only; no policy decision derives from them |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-08-01 | Denial of Service | Malformed/partial state.json crashes probeLlamaFirewall on an unchecked type assertion | mitigate | Read with comma-ok `.(float64)` and return false on any missing/wrong-typed field; missing file returns false (the WR-02 fix applied to the TUI probe) |
| T-08-08-02 | Spoofing | A stale state.json names a PID that has been reused by an unrelated process, showing a green pip for a dead sidecar | accept | Pip is informational only; PID-reuse false-positive is low-risk for a local single-user dashboard; the authoritative liveness is the supervisor itself |
| T-08-08-03 | Denial of Service | PID liveness check blocks the 10s healthTick on a slow syscall | mitigate | The check is a local FindProcess + signal-0 (sub-ms); no network, no IPC, no file scan; bounded work in the render-adjacent tick |
| T-08-08-04 | Information Disclosure | state.json fields beyond pid leak into the TUI | accept | Only the boolean liveness result is surfaced as a pip; no state.json content is rendered |
</threat_model>
