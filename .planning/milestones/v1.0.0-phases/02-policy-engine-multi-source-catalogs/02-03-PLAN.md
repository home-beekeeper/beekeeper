<objective>
Add three pure policy functions: network egress allowlist + size limits (PLCY-05), multi-turn exfiltration detection via Shannon entropy + base64 accumulation (PLCY-06), and the behavioral baseline frequency-deviation engine (PLCY-07). All in `internal/policy`, all pure. The `BaselineCounters` type defined here is the contract the Wave 2 `internal/baseline` store (Plan 06... actually Plan handled in Wave 2/3) loads and persists.

Purpose: These three policies share the "caller pre-resolves all inputs, policy stays pure" pattern. Co-located because exfil detection needs the rolling-window data the baseline engine produces — keeping the pure types together avoids a cross-package dependency cycle. No I/O here; the baseline persistence layer consumes `policy.BaselineCounters`.

Output: `internal/policy/egress.go`, `exfil.go`, `baseline.go` + tests. `BaselineCounters` is exported so the baseline store (later wave) marshals it to disk.
