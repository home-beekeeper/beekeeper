<objective>
Make fuzz testing a CI release gate for the policy engine (FuzzEvaluate) and the catalog parser (FuzzParseCatalogFile), extend the embedded selftest corpus with multi-source corroboration fixtures, and add an end-to-end integration test for a two-source corroborated block. This is the Phase 2 self-defense non-negotiable from CLAUDE.md ("Phase 2 is when fuzz tests become release-gating, not backlog").

Purpose: The corroboration engine and the catalog parser both process untrusted, attacker-influenceable input (tool-call JSON, catalog feed bytes). A panic on malformed input on the hot path is a fail-open vector. Fuzzing in CI catches these before release. The selftest corpus extension ensures the new corroboration semantics have a runtime sanity check that ships in the binary.

Output: `internal/policy/fuzz_test.go`, `internal/catalog/fuzz_test.go` (both `//go:build fuzz`), an extended `internal/check/corpus/fixtures.json` + any `selftest.go` adjustment, `internal/check/integration_test.go`, and a CI job in `.github/workflows/ci.yml`.
