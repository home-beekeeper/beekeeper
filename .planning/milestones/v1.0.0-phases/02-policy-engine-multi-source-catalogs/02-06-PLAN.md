<objective>
Deliver the release-age policy (PLCY-02) and lifecycle-script policy (PLCY-03) as the standard split: PURE policy functions in `internal/policy` (receiving caller-resolved inputs) plus their I/O adapters in `internal/catalog` (registry HTTP fetchers + a 24h publish-timestamp cache). These two policies fire independently of catalog corroboration.

Purpose: Release-age blocks freshly-published (potentially malicious) packages; lifecycle-script policy denies install-time code execution unless allowlisted. Both must fail CLOSED when registry data is unavailable. The pure/adapter split keeps `internal/policy` side-effect-free per the absolute project constraint.

Output: `internal/policy/release_age.go` + `lifecycle.go` (+ tests), `internal/catalog/registry.go` + `age_cache.go` (+ tests). Depends on Plan 01 (the Decision shape is stable from Phase 1; rule-ID constants for these rules are added in these files). Wiring into the hook handler is Plan 08.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| registry API → adapter | Untrusted JSON; a malicious mirror could backdate a fresh package's publish time to evade the age check |
| adapter → pure policy | Adapter must signal missing on failure so the pure layer fails closed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-06-01 | Spoofing | backdated publish timestamp | accept | Registries are the authoritative source over TLS; defending against a registry lying about its own publish times is out of v1 scope (documented). A compromised registry is a broader supply-chain failure |
| T-02-06-02 | Denial of Service | registry unavailability → fail closed blocks everything | mitigate | Missing:true cache entry written on error prevents repeated hammering; the block is correct (fail-closed per PLCY-02) but cached so it is not also a self-inflicted DoS on the registry |
| T-02-06-03 | Tampering | hidden I/O in pure policy layer | mitigate | EvaluateReleaseAge/EvaluateLifecycle have purity tests; all HTTP/time lives in the catalog adapters |
| T-02-06-04 | Elevation of Privilege | lifecycle script execution at install | mitigate | Default-deny preinstall/install/postinstall unless allowlisted; registry-check failure fails closed (blocks the install) |
</threat_model>
