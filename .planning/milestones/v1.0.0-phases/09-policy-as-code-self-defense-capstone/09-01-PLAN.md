<objective>
Build `internal/policyloader` — the I/O-side package that loads, validates, and dry-run-tests declarative JSON policy files (CODE-01..04). This package is the ONLY home for policy-file I/O; the pure `internal/policy` engine is never modified. The loader parses `policies/*.json` into typed rules, an enum/field validator rejects any execution-surface (url/exec/unknown rule_type/unknown schema_version), and a dry-run harness converts a policy file's rules into the inputs `policy.Evaluate` already accepts and reports the resulting Decision.

Purpose: Make policy version-controllable, testable, and validatable without breaking the pure-engine constraint that the hook handler, gateway, and Sentry all depend on.
Output: New package `internal/policyloader` with loader/validate/test source, white-box tests, and six testdata fixtures (valid + adversarial). This is Wave 0 for the rest of the phase — the package and its fixtures do not exist yet.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| developer/project repo → policyloader | Policy JSON files are attacker-influenceable (committed by a compromised agent or supply-chain) and crossed when LoadPolicyFile parses them |
| policyloader → policy engine | Only typed, validated data crosses; no raw file content reaches the pure engine |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Tampering | LoadPolicyFile (adversarial policy file smuggling exec) | mitigate | PolicyRule struct has no url/exec field; json.Decoder with DisallowUnknownFields rejects smuggled keys; ValidateSchema rejects Action=="exec" (V5/V10) |
| T-09-02 | Tampering | ValidateSchema (unknown rule_type silently skipped) | mitigate | rule_type is an enum; default switch branch is an error, never a skip; all errors collected and surfaced |
| T-09-03 | Elevation of Privilege | internal/policy purity | mitigate | All I/O lives in policyloader; loader converts to engine inputs before Evaluate — no os/io/net import added to internal/policy (CLAUDE.md absolute constraint) |
| T-09-04 | Denial of Service | LoadPolicyFile reading an unbounded file | accept | Policy files are local, developer-authored, and small; the existing 1MB stdin cap governs tool-call input separately. Bounded decode can be added if files grow — low risk for local committed files |
</threat_model>
