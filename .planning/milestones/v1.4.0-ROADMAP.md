# Milestone v1.4.0: Adjudicated Corpus (Local Loop)

**Status:** ✅ SHIPPED 2026-06-15
**Phases:** 22–25
**Total Plans:** 12 (3 per phase)

## Overview

Capture the **moat** — incidents with a confirmed ground-truth outcome attached — from the first run on a single, offline machine; wire confirmed adjudications into First Responder; and freeze the push-envelope wire format **without standing up any transport**. Scope = `beekeeper-corpus-milestone-prd.md` §3 (local loop only); the PRD's §6 (v1.1–1.9 org self-host aggregation + push) and §7 (v2.0 community shared feed) are deferred to future milestones.

The moat is the **outcome layer** (the confirmed `true_label` and how it was established): behavior + decision are cheap to reproduce, but a confirmed outcome cannot be retrofitted onto incidents already discarded — so the schema captures it from the first offline write, on one machine, with zero network. Enforcement stays local / offline / fail-closed in every version; the corpus loop and (future) push are off the hot path. The pure libraries (`internal/policy` / `nudge` / `pkgparse`) stayed I/O-free throughout.

Zero new dependencies (Go stdlib only: `ed25519`, `hmac`+`sha256`, `O_APPEND` NDJSON). A new impure `internal/corpus` package consumes the pure `internal/policy` + `internal/sentry`.

## Milestone Summary

**Key Decisions:**

- The moat is the **outcome layer** captured from the first offline write — non-retrofittable; distribution propagates the moat but is not the moat (built/hardened after launch).
- **Enforcement stays local / offline / fail-closed** in every version; the corpus loop and (future) push are off the `beekeeper check` hot path; `internal/policy` stays I/O-free.
- **`auto_purge` is unrepresentable** in a pushable envelope (compile-time guard, not a runtime string check) — the blast-radius guard baked in from the first record.
- OQ-1 `downstream_clean` = 30-day configurable window; OQ-2 `scan` `cluster_id` = per-package stable key; OQ-3 adjudicator lifecycle = bounded batch pass inside `runCatalogsSync` (OS-scheduler daemon, not a long-lived goroutine).
- Local overlay entries are **unsigned** (CTLG-07 anti-poisoning) → they escalate allow→warn, never block on their own.

**Issues Resolved (during the milestone):**

- ADJ-01 benchmark defect (global StateDir vs threaded cacheDir; nudge subprocess in timed loop) — caught by orchestrator post-wave gate, fixed before verify.
- CR-01 cross-process salt TOCTOU race — salt moved to its own `O_CREATE|O_EXCL` file.
- Config-merge prod bug (`cfg.Corpus.Enabled` always false) — caught by the Phase-24 synthetic E2E gate; `mergeCorpus`/`mergeAutoQuarantine` added.
- Two Phase-25 false-confidence launch gates (LAUNCH-01/03 assertions) — caught by code review; rewritten to genuinely fail on regression.
- **FRB-05 enforcement BLOCKER** (milestone audit) — the local overlay was written but never read by any production enforcement path; fixed via quick task `260615-ky4` (`handler.go` now uses `NewMultiIndexWithOverlay`; allow→warn escalation; RunCheck-level regression tests).

**Issues Deferred / Carried Forward:**

- Local overlay wiring for the MCP gateway / scan / watch surfaces (the `beekeeper check` hot path is wired; the others still use the no-overlay `NewMultiIndex`).
- Design question: should a confirmed-malicious overlay entry block (not just warn)? Would require signing local overlay entries — an anti-poisoning tradeoff.
- PRD §6 (org self-host aggregation + push, v1.1–1.9) and §7 (community shared feed, v2.0) — future milestones; the wire format is frozen and emitted-in-shape, so transport is wiring not migration.
- IPv6 bare-address `behavior_signature_hash` normalize quirk — accepted at the Phase 22 freeze; fix needs a `CorpusSchemaVersion` bump.

**Technical Debt Incurred:**

- STORE-02: `audit.NewMultiSinkWithCorpus` (the designed fan-out seam) has no production call site — corpus records reach the file via the direct `writeCorpusRecordDirect`/`AppendCorpusRecordLine` path instead. Records are written correctly; the fan-out architecture is just unused.
