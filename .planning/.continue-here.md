# Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Interval default vs clamp-floor coupling | Changing the catalog-sync DEFAULT without lowering the clamp FLOOR silently clamps it back (a 2h default under a 5h floor became 5h). | advisory | Any future `catalog_sync` interval change must keep `default` inside `[catalogSyncMinInterval, catalogSyncMaxInterval]`. Two paths exist: `parseClampCatalogSyncInterval` (clamps) and the TUI `persistCatalogSyncInterval` (validate-before-write, REJECTS out-of-range). Update both bounds + both test paths. |
| Public version != internal milestone | The public product is **v1.0.0** (single changelog entry); internal GSD tags v1.0.0/v1.2.0/v1.3.0 are NOT public releases. | advisory | Do NOT re-add `content/changelog/v1.2.0` or `v1.3.0` pages, and do NOT bump the site version badge (site-header/footer) past v1.0.0. The repo is unpushed, so keep "published/signed release" copy de-asserted. |
| GSD phase-number resolver regression | This project's resolvers mis-map bare phase numbers to archived milestone dirs and corrupt STATE frontmatter. | advisory | Drive any GSD phase op with explicit paths; hand-reconcile STATE/ROADMAP/REQUIREMENTS after `state.*`/`phase.complete`. |
